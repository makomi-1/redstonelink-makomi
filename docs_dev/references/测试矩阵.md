## 早期开发小矩阵（快速回归）

说明：
- 目标是数量少但覆盖关键冲突面。
- 红石算法改写类（Alternate Current / Carpet）默认分开跑。
- 核心栈默认固定启用：Lithium + FerriteCore + ModernFix + Krypton。
- 观测栈默认固定启用：Spark（用于 TPS/MSPT 与热点采样，不作为功能兼容阻断项）。
- 客户端口径：基线客户端 = Fabric Loader + Fabric API + 本模组。
- 运行口径：单机属于集成服，可用于快速回归；版本冻结前仍需补一轮独立服务端 + 外部客户端验证。

| ID | 服务端模组栈 | 客户端模组栈 | 重点验证 | 通过标准 |
|---|---|---|---|---|
| E0 | 原版 + 本模组 + Spark | 基线客户端（仅本模组） | 基线行为 | 锁定问题不复现，向上传播正确 |
| E1 | 核心栈（Lithium/FerriteCore/ModernFix/Krypton） + Spark | 基线客户端（仅本模组） | 启动与基础行为稳定性 | 启动成功，`/redstonelink diag lithium` 无关键异常 |
| E2 | E1 + Alternate Current | 基线客户端（仅本模组） | 红石路径改写兼容（AC） | 顶面核心粉切换与传播与 E1 一致 |
| E3 | E1 + Carpet | 基线客户端（仅本模组） | 红石规则改写兼容（Carpet） | 行为一致，关键规则切换后无漂移 |
| E4 | E1 + C2ME + VMP | 基线客户端（仅本模组） | 区块并发时序稳定性 | 高频触发无卡死或错态 |
| E5 | E1 + C2ME + VMP + Noisium | 基线客户端（仅本模组） | 高并发与调度混合压力 | 2 tick 周期压力下无异常锁定 |
| E6 | E1 | 扩展客户端（Sodium + Iris） | 客户端渲染与状态同步 | 方块状态显示与服务端一致 |
| E7 | E5 | 扩展客户端（Sodium + Iris + Entity Culling + More Culling） | 全链路早期集成 | 行为正确且无渲染不同步 |

建议执行顺序：`E0 -> E1 -> E2/E3 -> E4/E5 -> E6/E7`。

## 持续开发扩展矩阵（版本冻结前）

固定基线层：本模组 + Lithium + FerriteCore + ModernFix + Krypton + Spark。

- 维度 A（红石算法层）：A0 无改写 / A1 +Carpet / A2 +Alternate Current
- 维度 B（并发区块层）：B0 无增强 / B1 +C2ME+VMP / B2 +C2ME+VMP+Noisium
- 维度 C（客户端渲染层）：C0 基线客户端 / C1 +Sodium+Iris / C2 +Sodium+Iris+Entity Culling+More Culling

组合规模：`A(3) x B(3) x C(3) = 27 组`

执行分级：
- 日常开发（提交前）：跑 9 组抽样（覆盖每个维度至少一次）。
- 版本冻结前：跑全量 27 组。
