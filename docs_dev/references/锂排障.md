# Lithium 兼容排障手册

> 注：当前 `release/no-mixin` 分支已移除 Lithium Mixin 注入与 `/redstonelink diag lithium` 命令，本文中的注入命中与异常码判定仅作历史参考。

## 1. 目标

用于快速判断：是“没有走锂路由”、还是“路由命中但动态注入未执行”、还是“包/环境不一致”。

## 2. 三步定性流程

1. 看路由日志（`RedstoneLink/MixinRoute`）。
2. 进世界后复现场景，并观察 `redstonelink` 相关日志是否出现兼容异常。
3. 比对构建包 SHA256，确认外部实例使用的是当前构建产物。

## 3. 关键判定

- `applyLithiumMixin=true`：说明已走锂注入路由。
- `anomalyCode=RL-LI-001`：关键动态方法签名缺失。
- `anomalyCode=RL-LI-002`：动态路径命中总数为 0（高概率注入未真正执行）。
- `anomalyCode=RL-LI-003`：动态路径命中分布异常（可能存在行为漂移）。

## 4. 常见误区

1. 版本号一致不等于运行态一致（可能加载了旧 jar 或不同模组组合）。
2. `require=0` 会防崩，但也可能静默失效。
3. PowerShell 参数不加引号会被误解析，导致诊断日志与预期不符。

## 5. 日志定位关键词

- `RedstoneLink/MixinRoute`
- `RedstoneLink/Diag`
- `RL-LI-001`
- `RL-LI-002`
- `RL-LI-003`
- `InvalidInjectionException`

## 6. 复现记录模板

- 运行环境：MC/Fabric/Fabric API/Lithium 版本
- 模组列表：与本地基线差异
- 路由日志：是否 `applyLithiumMixin=true`
- 两次 `diag` 输出：完整粘贴
- 包哈希：工作区构建包 vs 外部 mods 包
- 现象：锁定/切换/向上传播是否异常
