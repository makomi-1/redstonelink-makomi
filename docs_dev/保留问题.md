### 持久化键与网络协议 ID 已硬切，旧版本不兼容
- 风险描述：
1. 当前已硬切到新契约（持久化键 `sourceSerial/targetSerials`、协议 ID `open_triggersource_pairing`），旧版本存档与旧端联机不再兼容。
2. 若后续需要回滚到旧版本，可能出现旧端无法识别新数据/新协议的问题。
- 规避方案：
1. 当前项目按“仅内部调用、单环境同版本”约束运行，统一使用同一版本客户端与服务端。
2. 如未来存在外部分发需求，再补“双读写 + 双协议 + 迁移回归”的兼容层。

### 命令激活跨区块提示缺失（无玩家上下文）
- 风险描述：
1. `activate triggersource` 属于命令触发路径，当前无玩家上下文时不会投递跨区块接管提示。
2. 与玩家交互触发（按钮/拉杆/遥控器）的提示行为不一致，可能造成“是否生效”的误判。
- 规避方案：
1. 当前阶段保留现状，命令激活验收以目标状态变化与结果汇总为准。
2. 后续可补充服务端提示投递策略（执行者回执、附近玩家广播或可配置日志）。

### 命令层解析结果二次包装可读性问题
- 风险描述：
1. `ModCommands.parseTargetSerials` 会把 `SerialParseUtil.TargetParseResult` 再包装为本地 `TargetParseResult`，阅读时容易被误解为重复构造或冗余逻辑。
2. 当后续字段扩展时，若两处 record 未同步更新，存在维护遗漏风险。
- 规避方案：
1. 当前阶段保留现状（命令层模型隔离）。
2. 后续可评估是否统一直接使用 `SerialParseUtil.TargetParseResult`，或在命令层补充明确注释说明“隔离映射”的意图。

### 跨区块通信无距离上限
- 风险描述：
1. 当前跨区块链路不按距离裁剪，远距离或跨维目标也可直接联动，误配置时影响面更大。
2. 当目标集合过大时，强加载/中继接管压力会随触发范围放大。
- 规避方案：
1. 现阶段通过白名单/预设与 `server.maxTargetsPerSetLinks` 控制联动范围。
2. 后续如需增强可控性，可补充“通信距离上限”配置与超限提示。

### 发射器跨区块接管提示缺失
- 风险描述：
1. 发射器已接入跨区块派发链路，但无玩家上下文时不会触发提示，易被误判为“未生效”。
2. 与按钮/拉杆/遥控器的提示行为不一致，排障与联调成本上升。
- 规避方案：
1. 当前阶段保留现状，跨区块是否生效以目标状态变化/链路结果为准进行验收。
2. 后续如需统一体验，可补充“无玩家上下文提示投递策略”（附近玩家广播或可配置服务端日志）。

3. 第三方更高优先级 mixin 可能引发行为漂移。
4. 同目标注入顺序变化会改变最终执行语义。
- 规避方案：
