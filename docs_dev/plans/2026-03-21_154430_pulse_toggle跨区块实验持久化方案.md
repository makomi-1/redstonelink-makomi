# pulse_toggle跨区块实验持久化方案

生成时间：2026-03-21 15:44:30
文件名：2026-03-21_154430_pulse_toggle跨区块实验持久化方案.md

## 任务背景

当前跨区块调度中，`sync` 已具备“最新状态不限时持久化”的能力，但 `pulse/toggle` 仍混用 `ACTIVATION` 队列语义：

1. `pulse/toggle` 虽可进入持久队列，但数据结构按同 key 覆盖，无法稳定表达事件语义；
2. 来源区块卸载会通过失效重算撤销 `pulse/toggle` 的贡献，不适合作为“历史事件延迟投递”玩法；
3. 需要将 `sync` 与 `pulse/toggle` 分离为“状态型持久同步”与“事件型实验玩法”两套语义，并明确默认值。

本次确认后的目标是：

1. `sync` 中继持久化默认开启；
2. `pulse/toggle` 的普通 TTL relay 独立默认关闭；
3. `pulse/toggle` 的无限期投递作为实验性玩法，独立默认关闭；
4. 保持内部语义统一为 `triggerSource/core`，不引入旧术语输出。

## 方案详情

### 现状分析

1. 现有跨区块队列由 `CrossChunkDispatchQueueSavedData` 持久化，键模型为 `source/target/dispatchKind`，适合 `sync` 的“最新状态”语义，不适合 `toggle` 的奇偶事件语义。
2. `CrossChunkDispatchService` 当前仅区分 `ACTIVATION` 与 `SYNC_SIGNAL`，`pulse/toggle` 共用一套 TTL 与 upsert 行为。
3. `ActivatableTargetBlockEntity` 已具备“无来源序号时按轻量事件处理、不进入来源桶”的入口，可作为实验性事件投递的落点。
4. 生命周期回放与来源失效路径当前会统一清理 `sync/pulse/toggle` 三类贡献，需要收口，否则实验性延迟事件会被来源卸载抵消。

### 技术方案

1. 配置层扩展
   - 保留 `crosschunk.syncSignalPersistent=true` 作为默认状态型持久同步。
   - 新增 `crosschunk.activation.pulse.relay.enabled=false`。
   - 新增 `crosschunk.activation.pulse.ttlTicks`。
   - 新增 `crosschunk.activation.pulse.persistentExperimental=false`。
   - 新增 `crosschunk.activation.toggle.relay.enabled=false`。
   - 新增 `crosschunk.activation.toggle.ttlTicks`。
   - 新增 `crosschunk.activation.toggle.persistentExperimental=false`。

2. 队列结构拆分
   - 将现有 `ACTIVATION` 拆分为显式 `PULSE_EVENT` 与 `TOGGLE_EVENT`。
   - 旧存档中的 `ACTIVATION` 队列项在加载时按 `activationMode` 迁移为新 kind，保证兼容。

3. `pulse` 语义
   - TTL 模式：仅在 `pulse.relay.enabled=true` 时允许入队，使用 `pulse.ttlTicks` 过期。
   - 实验模式：`pulse.persistentExperimental=true` 时允许无限期待投递。
   - 聚合策略：同一 `triggerSource -> core` 在目标离线期间合并为“一次待投递脉冲”，不记录完整脉冲日志。

4. `toggle` 语义
   - TTL 模式：仅在 `toggle.relay.enabled=true` 时允许入队，使用 `toggle.ttlTicks` 过期。
   - 实验模式：`toggle.persistentExperimental=true` 时允许无限期待投递。
   - 聚合策略：同一 `triggerSource -> core` 在目标离线期间累计“净奇偶”；若奇偶归零则删除 pending。

5. 投递落点
   - `sync` 继续按来源贡献语义写入来源桶并参与失效重算。
   - `pulse/toggle` 的队列投递改走轻量事件入口，不进入来源桶，避免后续来源卸载把已投递事件回滚。

6. 失效收口
   - `SOURCE_INVALIDATION` 收口为仅影响 `sync`。
   - `pulse/toggle` 作为事件语义，不再由来源卸载回滚。

### 影响范围

1. 配置解析与默认配置模板：`RedstoneLinkConfig`
2. 跨区块持久队列模型与兼容加载：`CrossChunkDispatchQueueSavedData`
3. 派发决策、TTL/持久化判定、重试策略：`CrossChunkDispatchService`
4. 目标端投递与失效语义：`ActivatableTargetBlockEntity`
5. 生命周期回放与内部事件投影：`InternalDispatchDeltaEvents`、`InternalDispatchDeltaProjector`、`LinkNodeLifecycleDispatchEvents`
6. 使用说明与测试：`docs_dev/使用说明.md`、相关测试文件

## 语义对齐说明

1. 命名仅使用 `triggerSource/core`，与 `LinkNodeType.TRIGGER_SOURCE/CORE` 保持一一映射。
2. 输入输出方向保持 `triggerSource -> core`，不允许反向投递。
3. 新增配置注释、文案、说明不泄漏 `button/source/target` 旧术语。
4. 兼容层仅做“读旧写新”：读取旧 `ACTIVATION` 数据，写回使用新 kind 与新配置语义。
5. 残余风险：
   - `pulse` 采用“单次补发”而不是完整历史脉冲重演；
   - `toggle` 采用“净奇偶”而不是完整历史事件日志；
   - 这两项均为刻意取舍，用于平衡玩法价值与复杂度。

## 原子步骤清单

### 步骤 1：扩展跨区块配置模型
- **操作对象**：`RedstoneLinkConfig`
- **具体动作**：新增 `pulse/toggle` 的 TTL relay 开关、TTL、实验性持久化开关与默认模板注释。
- **预期结果**：默认策略可由配置完整表达。
- **关键里程碑**：是

### 步骤 2：拆分 activation 队列 kind 并补兼容迁移
- **操作对象**：`CrossChunkDispatchQueueSavedData`
- **具体动作**：新增 `PULSE_EVENT/TOGGLE_EVENT`，对旧 `ACTIVATION` 持久项做读档迁移。
- **预期结果**：`pulse/toggle` 可按各自语义独立建 key。
- **关键里程碑**：是

### 步骤 3：实现 pulse/toggle 独立入队判定
- **操作对象**：`CrossChunkDispatchService`
- **具体动作**：根据新配置分别决定跳过、TTL 入队或无限期入队。
- **预期结果**：`sync` 默认开，`pulse/toggle` 普通 relay 默认关，实验模式默认关。
- **关键里程碑**：是

### 步骤 4：实现 pulse one-shot 与 toggle parity 聚合
- **操作对象**：`CrossChunkDispatchQueueSavedData`、`CrossChunkDispatchService`
- **具体动作**：`pulse` 合并为一次待投递脉冲，`toggle` 记录净奇偶并支持归零删除。
- **预期结果**：事件型玩法具备最小可行聚合语义。
- **关键里程碑**：是

### 步骤 5：调整目标端投递语义
- **操作对象**：`ActivatableTargetBlockEntity`
- **具体动作**：为实验性 `pulse/toggle` 提供轻量事件投递路径，不写入来源桶。
- **预期结果**：已投递事件不再被来源卸载回滚。
- **关键里程碑**：是

### 步骤 6：收口来源失效与生命周期回放
- **操作对象**：`InternalDispatchDeltaEvents`、`InternalDispatchDeltaProjector`、`LinkNodeLifecycleDispatchEvents`
- **具体动作**：将 `SOURCE_INVALIDATION` 实际影响限定为 `sync`，保持 direct path/queue path 一致。
- **预期结果**：事件语义与状态语义彻底分离。
- **关键里程碑**：否

### 步骤 7：更新说明与补全回归测试
- **操作对象**：`docs_dev/使用说明.md`、测试文件
- **具体动作**：补充默认值、实验性玩法说明、兼容迁移与自动化测试。
- **预期结果**：文档与回归证据完整。
- **关键里程碑**：是

## 预期结果

1. `sync` 保持默认“最新状态持久同步”。
2. `pulse/toggle` 默认不做跨区块 relay；仅在显式开启时才参与 TTL 或实验性无限期投递。
3. `pulse` 支持实验性“一次性潜伏脉冲”。
4. `toggle` 支持实验性“净奇偶延迟投递”。
5. 行为差异、默认值与风险均可通过配置与文档明确表达。
