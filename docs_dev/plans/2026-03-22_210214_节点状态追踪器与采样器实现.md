# 节点状态追踪器与采样器实现

生成时间：2026-03-22 21:02:14
文件名：2026-03-22_210214_节点状态追踪器与采样器实现.md

## 任务背景
- 当前命令侧仅支持节点静态信息与部分 `core` 运行态查询，缺少统一的“当前状态探针”与“历史采样”能力。
- bench、诊断和后续物品状态面板都需要可复用的节点状态读取入口。
- 本轮目标是直接实现首版最小可用的状态追踪器与采样器，而不是只停留在方案文档。

## 方案详情

### 现状分析
- `ModCommands` 已有 `/redstonelink node get`，但运行态输出仅覆盖在线 `core`。
- `ActivatableTargetBlockEntity` 已暴露 `active / resolvedOutputPower / configuredMode / effectiveMode / resolvedStrength / maxSources`。
- `SyncReplaySourceBlockEntity` 已保存最近一次 sync 回放快照，`LinkSyncEmitterBlockEntity` 额外保存最近一次已观察输入强度。
- 项目已有 `ServerTickEvents` 生命周期注册点，适合挂载轻量级按 tick 采样服务。

### 技术方案
- 新增 `NodeRuntimeProbe`：统一读取 `core` 与 `sync triggerSource` 的当前瞬时状态。
- 新增 `NodeStateTraceService`：按 `type + serial` 动态挂载采样器，使用 ring buffer 保存最近 N 条样本。
- 命令入口挂到 `/redstonelink node trace ...`，首版支持：
  - `mount <type> <serial> [every] [capacity]`
  - `latest <type> <serial>`
  - `read <type> <serial> [limit]`
  - `unmount <type> <serial>`
  - `list`
- 首版支持对象：
  - `core`
  - 在线可识别的 `sync triggerSource`
- 采样粒度首版按 tick 实现，`slot` 统一输出 `0`；`core` 额外暴露 authority time key。
