# 命令测试模式去除bench额外玩家依赖

生成时间：2026-03-22 18:16:36
文件名：2026-03-22_181636_命令测试模式去除bench额外玩家依赖.md

## 任务背景
当前 bench 在 dedicated server 上运行时，需要额外在线玩家作为命令上下文，因为 `redstonelink place` 与 `redstonelink link set` 强依赖 `source.getPlayer()`。这会对性能测试引入额外玩家干扰，不利于做更稳定的自动化压测。

## 方案详情

### 现状分析
- `place setblock/fill` 与 `link set` 入口都直接要求 `ServerPlayer`，无玩家源时返回 `player_only`。
- `place` 实际只依赖执行世界；`link set` 主要依赖执行世界、写入权限校验，以及玩家背包物品快照同步。
- bench 脚本当前也默认强制要求 `-AsPlayer`，无法直接利用控制台/RCON 命令源。

### 技术方案
- 新增配置项 `server.command.benchmarkMode.enabled`，默认 `false`。
- 开启后，仅允许 `place setblock/fill` 与 `link set` 在无玩家源时继续执行。
- 世界上下文统一从 `CommandSourceStack.getLevel()` 读取，不再绑定 `player.serverLevel()`。
- 玩家背包物品快照同步仅在命令源确实是玩家时执行。
- 写入控制权限提升改用 `source.hasPermission(...)`，避免控制台兼容路径额外提权。
- bench 脚本放宽 `-AsPlayer` 强制要求：未提供时不再本地阻断，而是直接以控制台/RCON 源执行；若服务器未开启 benchmark mode，则仍会收到命令侧 `player_only` 报错。
