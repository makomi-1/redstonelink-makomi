# 恢复销毁打标回调

## 任务背景
创造模式下将可配对节点物品丢入岩浆后未自动退役。定位显示 `LinkNodeRetireEvents` 的 `DISCARDED` 分支依赖 `discardedByDamage`，而该值来自预写入标记消费；当前运行中打标入口未命中，导致逻辑在低年龄 `DISCARDED` 分支提前返回。

## 方案详情

### 现状分析
- `ENTITY_UNLOAD` 中对 `DISCARDED` 有低年龄早退，需依赖伤害打标才能判定为销毁。
- 历史提交曾通过 `PairableBlockItem#onDestroyed(ItemEntity)` 写入伤害打标。
- 后续提交移除了该回调，造成 `markDamageDiscard` 无调用入口，形成回归。

### 技术方案
- 在 `PairableBlockItem` 恢复 `onDestroyed(ItemEntity)` 重写。
- 在该回调中调用 `LinkNodeRetireEvents.markDamageDiscard(itemEntity)`，随后调用 `super.onDestroyed(itemEntity)`。
- 保持现有 `LinkNodeRetireEvents` 的 `DISCARDED` 判定不变，避免扩大行为面。

### 影响范围
- `src/main/java/com/makomi/item/PairableBlockItem.java`
