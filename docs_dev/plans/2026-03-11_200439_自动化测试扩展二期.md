## 任务背景

## 方案详情

### 现状分析
- 退役、清链路与审计快照的关键路径缺少针对性测试。
- 默认门禁依赖 stable-core，需要保证新增测试不引入高成本依赖。

### 技术方案
1. 新增序列号治理与退役链路回归，覆盖退役后清链路与分配跳过逻辑。
2. 新增清链路行为测试，确保双向索引一致。
3. 新增审计快照测试，覆盖缺失端点与统计计数。
4. 所有新增用例标记为 `stable-core`，不引入 API 测试。
