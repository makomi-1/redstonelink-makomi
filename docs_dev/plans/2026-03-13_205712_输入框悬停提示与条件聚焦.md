# 输入框悬停提示与条件聚焦

生成时间：2026-03-13 20:57:12
文件名：2026-03-13_205712_输入框悬停提示与条件聚焦.md

## 任务背景
配对界面存在输入体验问题：  
1) 输入框在空内容场景有“跳动”感；  
2) 输入提示过长，不利于快速阅读。

## 方案详情

### 现状分析
- 输入框在初始化阶段存在默认焦点处理，易导致空输入场景视觉跳动。
- 输入框提示内容承载了规则与示例，长度偏长。

### 技术方案
- 将自动聚焦策略改为“仅非空内容时自动聚焦”。
- 将输入框 hint 改为短提示。
- 新增输入框悬停 tooltip，展示完整规则与示例。

### 影响范围
- `src/client/java/com/makomi/client/screen/AbstractMultiPairingScreen.java`
- `src/main/resources/assets/redstonelink/lang/zh_cn.json`
- `src/main/resources/assets/redstonelink/lang/en_us.json`
- `docs_dev/使用说明.md`

## 原子步骤清单

### 步骤 1：条件自动聚焦
- **操作对象**：`AbstractMultiPairingScreen`
- **具体动作**：仅在初始化文本非空时执行 `setInitialFocus(serialInput)`。
- **预期结果**：空输入场景不抢焦点，非空场景保持编辑便利。
- **关键里程碑**：是

### 步骤 2：输入框悬停 tooltip
- **操作对象**：`AbstractMultiPairingScreen`
- **具体动作**：增加输入框区域 hover 检测，渲染规则与示例两行 tooltip。
- **预期结果**：完整提示可通过悬停查看。
- **关键里程碑**：是

### 步骤 3：文案与说明同步
- **操作对象**：`zh_cn.json` / `en_us.json` / `docs_dev/使用说明.md`
- **具体动作**：缩短 hint 文案，新增 tooltip 文案 key，并补充使用说明。
- **预期结果**：交互与文案一致。
- **关键里程碑**：否

## 预期结果
- 输入框空内容时不再出现跳动感；
- hint 保持简短可读；
- 通过输入框悬停可查看完整输入规则与示例。
