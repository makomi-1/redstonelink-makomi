## 任务背景
在不改变行为的前提下，推进代码命名统一：保留旧入口兼容，同时提供语义更清晰的新命名入口，供后续分批迁移。

## 方案详情

### 现状分析
- 语义层已有 `resolveTypeForRole` / `resolveCanonicalTypeForRole`，但命名未明确区分“兼容词表”和“严格词表”。
- 调用点已统一走语义门面，但命名仍可进一步清晰化。

### 技术方案
- 语义策略层新增：
  - `resolveCompatibleTypeForRole`（兼容词表）
  - `resolveStrictTypeForRole`（严格词表）
- 语义门面新增同名方法并保持旧方法委托到新方法，保证行为不变。
- 将配置层与 crosschunk 命令层调用切换到新命名方法。
- 补充测试验证新旧命名入口行为等价。
