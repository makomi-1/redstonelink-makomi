# P3-3 扇出诊断计数

生成时间：2026-03-21 12:24:11
文件名：2026-03-21_122411_P3-3扇出诊断计数.md

## 任务背景
当前 1:900 同步并发场景下，热点主要集中在邻居更新链路。为继续优化，需要先把“扇出是否重复、跨区块是否被跳过、去重是否命中”量化出来，作为后续 P3 收敛的决策依据。

## 方案详情

### 现状分析
- 二级扇出已统一到 `NeighborFanoutUtil`，但缺少运行时结构化计数。
- 实体侧已有时间粒度去重守卫 `shouldFanoutByResolvedOutput`，但没有命中统计。
- 目前只能通过 JFR 间接观察热点，无法直接从业务链路确认“发送量 vs 被抑制量”。

### 技术方案
- 在 `NeighborFanoutUtil` 增加轻量计数器（`LongAdder`）：
  - 扇出请求次数（调用入口）
  - 中心通知发送次数
  - 邻居通知尝试次数
  - 邻居通知实际发送次数
  - 跨 chunk 邻居跳过次数（未加载）
- 在 `ActivatableTargetBlockEntity` 的去重守卫增加命中计数：
  - 同时间粒度同输出值的抑制次数
- 提供统一快照与重置入口，供后续日志/命令/测试读取。
- 增加契约测试，确保计数接口与关键计数点存在，防止后续回归。

### 影响范围
- `src/main/java/com/makomi/util/NeighborFanoutUtil.java`
- `src/main/java/com/makomi/block/entity/ActivatableTargetBlockEntity.java`
- `src/test/java/com/makomi/block/LinkRedstoneDustCoreStateContractTest.java`
