# 批量命令覆盖确认与激活退役扩展

生成时间：2026-03-13 22:52:04
文件名：2026-03-13_225204_批量命令覆盖确认与激活退役扩展.md

## 任务背景
用户确认本轮实现以下能力：
1. 批量设置白名单。
2. 命令端批量激活（命令驱动开放入口）。
3. 批量退役节点。
4. 批量删关系（增量删）暂缓。
5. 所有“批量覆盖式”命令统一二次确认（`+confirm`）。

## 方案详情

### 现状分析
- `set_links` 已支持批量格式解析（`N`/`A:B` + `/`），但当前为直接执行、无确认门槛。
- `crosschunk whitelist` 仅支持单序号 `add/remove/list/clear`。
- `retire` 当前仅单序号命令，支持 `confirm`。
- 派发链路已具备批量可复用能力，可新增 `activate` 命令入口调用。

### 技术方案
1. `set_links` 批量覆盖确认
   - 命令树增加 `confirm` 分支。
   - 仅当解析后目标数量 > 1 时要求 `confirm`。
   - 保持清空（不带 targets）和单目标写入原有行为。
   - GUI 发出的批量 `set_links` 自动附带 `confirm`，避免现有交互回归。

2. `crosschunk whitelist set` 批量覆盖
   - 新增 `set <role> <type> <serials> [confirm]`。
   - 全量校验后执行覆盖（清空该 role+type 后写入新集合）。
   - 二次确认强制要求（无 `confirm` 仅预览与提示）。

3. `activate` 命令子模块
   - 新增子包命令注册器：`activate triggersource <source_serials> [mode]`。
   - `mode` 支持 `toggle|pulse`，默认 `toggle`。
   - 复用 `LinkedTargetDispatchService.dispatchActivation` 执行并输出汇总结果。

4. `retire batch` 命令子模块
   - 新增 `retire batch <type> <serials> [confirm]`。
   - 先全量校验（已分配且未退役），再批量执行。
   - 无 `confirm` 时仅返回预览与确认提示。

5. 文案与文档同步
   - 增加命令反馈文案 key（中英文）。
   - 在 `docs_dev/使用说明.md` 增补新命令与确认规则。
