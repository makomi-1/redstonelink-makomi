# place fill 确认策略统一

生成时间：2026-03-21 14:49:20  
文件名：2026-03-21_144920_place_fill确认策略统一.md

## 任务背景
- 现有 `place fill` 使用“仅替换非空气方块时拦截”的策略，`setblock` 也存在非空气替换确认链路。
- 目标是简化命令行为：取消非空气替换检测，`fill` 采用统一确认策略，`setblock` 不拦截。

## 方案详情

### 现状分析
- `fill` 与 `setblock` 都依赖非空气替换计数方法。
- `setblock` 通过待确认队列和 `/redstonelink place confirm` 完成二次确认。
- 语言文案与使用说明均围绕“非空气替换检测”描述。

### 技术方案
- 删除非空气替换计数逻辑与 `place confirm` 待确认链路。
- `place fill` 改为：非 `force` 且未追加尾缀 `confirm` 时统一拦截。
- `place setblock` 改为：不做确认拦截，直接执行。
- 同步更新中英文文案与使用说明，移除旧语义描述。

### 影响范围
- 命令实现：`ModCommands`
- 文案资源：`zh_cn.json`、`en_us.json`
- 使用说明：`docs_dev/使用说明.md`

## 原子步骤清单

### 步骤 1：移除空气检测与待确认链路
- **操作对象**：`ModCommands`
- **具体动作**：删除非空气替换计数、删除 `place confirm` 命令与缓存结构。
- **预期结果**：命令逻辑不再依赖非空气替换判断。
- **关键里程碑**：是

### 步骤 2：统一 fill 拦截规则
- **操作对象**：`ModCommands`
- **具体动作**：将 `fill` 改为非 `force` 且无尾缀 `confirm` 统一拦截。
- **预期结果**：`fill` 确认行为一致可预期。
- **关键里程碑**：是

### 步骤 3：同步文案与说明
- **操作对象**：多语言资源与使用说明文档
- **具体动作**：更新确认提示文案，清理“非空气替换才拦截”描述。
- **预期结果**：命令行为与文档一致。
- **关键里程碑**：否

## 预期结果
- `place fill` 只保留“尾缀 confirm/force”两种放行方式。
- `place setblock` 不再受确认拦截影响。
- 命令文案与使用说明不再包含非空气替换检测语义。
