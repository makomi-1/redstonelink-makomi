# 自动化测试扩展方案

生成时间：2026-03-11 12:44:38
文件名：2026-03-11_124438_自动化测试扩展方案.md

## 任务背景
当前自动化测试集中在稳定核心逻辑，默认测试门槛仅覆盖 `stable-core` 标签；扩展层与覆盖率尚未形成体系化约束。
本方案目标是在不影响主线构建速度的前提下，逐步补齐逻辑覆盖、兼容性回归与集成层验证，并明确“API 不做相关测试”的约束。

## 方案详情

### 现状分析
- 已存在 `stable-core` 测试门槛与 `testApiLegacy` 任务，但默认 gate 仅覆盖 `stable-core`。
- CI 的 L1 checks 会执行 `test` 并上传测试报告。
- 测试用例主要覆盖数据/配置/网络/方块实体内部逻辑。
- 覆盖率统计未接入，无法量化覆盖面。

### 技术方案
1. 保持 `stable-core` 作为“快速回归门槛”，确保 PR/Push 的构建速度。
2. 增加分层测试标签与任务（`integration` / `client` / `slow`），默认不进入主 gate，仅在手动或夜间触发。
3. 引入覆盖率统计（例如 Jacoco），先只统计不设阈值，再逐步设门槛。
4. 重点补齐：序列化/NBT 兼容、网络协议负面用例、关键流程集成测试。
5. 明确约束：不新增 API 相关测试，且不依赖 `com.makomi.api.v1.*`。

### 影响范围
- `build.gradle`（测试任务与覆盖率配置）
- `.github/workflows/build.yml`（CI 触发策略与报告上传）
- `src/test/java/**`（新增或扩展测试）
- `src/test/resources/**`（版本化 NBT fixture）

## 原子步骤清单

### 步骤 1：测试分层与任务拆分
- **操作对象**：`build.gradle`
- **具体动作**：新增 `testIntegration`、`testClient`、`testSlow` 任务并使用标签过滤；`test` 维持只包含 `stable-core`。
- **预期结果**：稳定门槛不变，扩展测试可按需触发。
- **关键里程碑**：是

### 步骤 2：扩展稳定核心单元测试
- **操作对象**：`src/test/java/com/makomi/{data,config,network,block/entity}`
- **具体动作**：补齐边界条件、非法输入、不可变返回、序列化回放等用例；确保 `stable-core` 时长可控。
- **预期结果**：核心逻辑回归更健壮。
- **关键里程碑**：否

### 步骤 3：引入版本化 NBT fixture 回归
- **操作对象**：`src/test/resources/fixtures/nbt/**`
- **具体动作**：存放旧版本 NBT 样例；新增参数化测试验证兼容逻辑与异常路径。
- **预期结果**：存档兼容性可验证、可回归。
- **关键里程碑**：否

### 步骤 4：网络协议负面用例补齐
- **操作对象**：`src/test/java/com/makomi/network/**`
- **具体动作**：覆盖空负载、非法序列号、字段缺失/越界等边界场景。
- **预期结果**：协议稳定性提升，防回归。
- **关键里程碑**：否

### 步骤 5：引入集成层验证（Fabric GameTest 优先）
- **操作对象**：`src/test/java/...` + `run/saves/test`（如需）
- **具体动作**：编写关键流程测试：配对/断开/激活/存档读写；标记 `integration`。
- **预期结果**：关键用户流程可自动回归。
- **关键里程碑**：是

### 步骤 6：接入覆盖率统计与报告上传
- **操作对象**：`build.gradle`、`.github/workflows/build.yml`
- **具体动作**：启用 Jacoco 报告输出（HTML/XML），CI 上传 artifacts；先不设置阈值。
- **预期结果**：覆盖面可量化，后续可设门槛。
- **关键里程碑**：是

### 步骤 7：CI 分层策略
- **操作对象**：`.github/workflows/build.yml`
- **具体动作**：PR/Push 仅跑 `stable-core`；手动/夜间触发 `integration/client/coverage`。
- **预期结果**：主线效率与扩展验证并存。
- **关键里程碑**：否

## 预期结果
- 稳定门槛不变、扩展层可按需触发。
- 覆盖率统计可持续观测。
- 关键流程拥有可复用的自动化回归路径。
- 明确保证：不增加 API 相关测试依赖。
