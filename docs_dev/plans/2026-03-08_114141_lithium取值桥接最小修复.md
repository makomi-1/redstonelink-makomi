# lithium取值桥接最小修复

生成时间：2026-03-08 11:41:41
文件名：2026-03-08_114141_lithium取值桥接最小修复.md

## 任务背景
已确认有锂环境下锂专用 mixin 注入生效，但“锁定无法切换”和“向上传播异常”问题仍存在。

## 方案详情

### 现状分析
- 现有锂 mixin 已桥接 state.is(this) 判定。
- lithium getPowerFromSide(...) 路径中仍直接读取 state.getValue(POWER)，未复用核心粉取值语义。

### 技术方案
- 仅修改 RedStoneWireBlockLithiumMixin：
  - 对 getPowerFromSide(...) 内 BlockState#getValue(Property) 增加 3 个 @Redirect（ordinal 0/1/2）。
  - 统一通过 helper 返回 wire-like 功率：核心粉走 LinkRedstoneDustCoreBlock.getTopAttachedWireSignal(state)，原版粉走 POWER。
- 不改动插件和无锂 mixin，保持最小变更面。

### 影响范围
- src/main/java/com/makomi/mixin/RedStoneWireBlockLithiumMixin.java

## 原子步骤清单

### 步骤 1：新增 getValue 重定向
- **操作对象**：RedStoneWireBlockLithiumMixin
- **具体动作**：在 getPowerFromSide(...) 添加 3 个 @Redirect 指向统一 helper
- **预期结果**：锂快路径下核心粉/原版粉取值语义一致
- **关键里程碑**：是

### 步骤 2：新增统一取值 helper
- **操作对象**：RedStoneWireBlockLithiumMixin
- **具体动作**：新增 getEquivalentWirePower(BlockState, Property<?>) 方法
- **预期结果**：避免重复逻辑，降低后续维护成本
- **关键里程碑**：否

### 步骤 3：编译验证
- **操作对象**：Gradle
- **具体动作**：执行 compileJava（可选补 emapJar）
- **预期结果**：补丁可编译并可打包
- **关键里程碑**：否

## 预期结果
在不扩大改动面的前提下，补齐 lithium 快路径中功率读取语义缺口，为外部有锂环境回归提供最小可测补丁。
