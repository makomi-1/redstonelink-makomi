# lithium兼容适配核心粉激活

生成时间：2026-03-07 23:49:00
文件名：2026-03-07_234900_lithium兼容适配核心粉激活.md

## 任务背景
在 WAYFARING 整包中定位到与 lithium 叠加时，核心粉出现“不能被物品激活/右键激活，但红石信号可触发”的问题。用户确认 redstonelink 新版本单独运行正常，因此判定为兼容问题而非功能回归。

## 方案详情

### 现状分析
- 当前核心粉兼容主要依赖 `RedStoneWireBlockMixin` 对 `calculateTargetStrength` 的中途 `Redirect`。
- lithium 在 `RedStoneWireBlock` 的 `getReceivedPower*` 分支做了优化注入，可能导致中途重定向路径不稳定或被旁路。
- 导致现象是玩家触发链已经到达 BE，但功率计算链没有稳定并入“顶面核心粉模拟输入”。

### 技术方案
采用“返回值兜底合并”的最小侵入方案：
1. 在 `calculateTargetStrength` 增加 `@Inject(at = @At("RETURN"), cancellable = true)`。
2. 在返回前统一执行 `max(原返回值, 顶面核心粉模拟输入)`。
3. 保留现有 `Redirect` 逻辑，兼容 vanilla 与非 lithium 路径。
4. 若实测仍有遗漏，再作为后续步骤补 `getReceivedPower@RETURN` 兜底。

### 影响范围
- `src/main/java/com/makomi/mixin/RedStoneWireBlockMixin.java`
- 构建验证（`compileJava`）

## 原子步骤清单

- **操作对象**：`RedStoneWireBlockMixin`

- **预期结果**：在 lithium 与 vanilla 分支下都能保留核心粉模拟输入
