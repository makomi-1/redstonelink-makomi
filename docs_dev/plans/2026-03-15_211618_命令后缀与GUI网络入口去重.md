# 命令后缀解析与GUI/网络入口去重

生成时间：2026-03-15 21:16:18
文件名：2026-03-15_211618_命令后缀与GUI网络入口去重.md

## 任务背景
当前命令层（ModCommands、RetireBatchCommandRegistry、CrossChunkCommandRegistry）存在重复的后缀解析逻辑；客户端 GUI（TriggerSource/Core 配对界面）与网络入口（RedstoneLinkClient）存在重复的入口分发与 set_links 命令构建逻辑。已确认目标是“去重但保持行为不变”，并保留 triggerSource/core 语义映射。

## 方案详情

### 现状分析
- 命令后缀解析在 3 处重复，且大小写策略不一致：
  - 普通确认后缀：大小写敏感 ` confirm`
  - 跨区块后缀：大小写不敏感 `resident/confirm`，并支持顺序无关
- GUI 两个子类重复拼接 `set_links` 命令与发送逻辑。
- RedstoneLinkClient 中按钮/核心入口分发在 hooks 与网络接收中重复。

### 技术方案
1. 抽取共享命令后缀解析工具，支持：
   - confirm-only（可配置大小写）
   - resident+confirm（可配置大小写，支持链式后缀剥离）
2. 在 AbstractMultiPairingScreen 中统一构建并发送 `set_links` / `clear_links`，子类仅提供来源类型语义。
3. 在 RedstoneLinkClient 中抽取统一的“按来源类型打开配对界面”入口，Hooks 与网络接收复用。

### 影响范围
- `src/main/java/com/makomi/command/**`
- `src/client/java/com/makomi/client/screen/**`
- `src/client/java/com/makomi/RedstoneLinkClient.java`
- 相关测试用例：命令契约、网络分发、客户端契约

## 原子步骤清单

### 步骤 1：抽取命令后缀解析工具
- **操作对象**：命令模块公共工具类
- **具体动作**：新增后缀解析结果模型与解析函数，覆盖 confirm-only 与 resident+confirm 两类场景
- **预期结果**：三处命令入口可复用同一解析逻辑
- **关键里程碑**：是

### 步骤 2：替换命令入口重复逻辑
- **操作对象**：ModCommands、RetireBatchCommandRegistry、CrossChunkCommandRegistry
- **具体动作**：删除局部解析记录类与重复函数，改为调用共享工具
- **预期结果**：行为保持一致，代码重复显著减少
- **关键里程碑**：是

### 步骤 3：抽象 GUI 统一发送入口
- **操作对象**：AbstractMultiPairingScreen、TriggerSourcePairingScreen、CorePairingScreen
- **具体动作**：将 `set_links` 与 `clear_links` 发送逻辑下沉到抽象父类，子类仅返回来源类型
- **预期结果**：两个子类不再维护重复发送逻辑
- **关键里程碑**：是

### 步骤 4：统一客户端入口分发
- **操作对象**：RedstoneLinkClient
- **具体动作**：提取按节点来源类型打开界面的方法，Hooks 与网络接收共用
- **预期结果**：入口分发逻辑单点维护，避免行为漂移
- **关键里程碑**：是

### 步骤 5：回归验证
- **操作对象**：测试任务
- **具体动作**：运行命令契约、网络分发与客户端契约相关测试
- **预期结果**：保证行为等价且无回归
- **关键里程碑**：是

## 预期结果
在不改变命令语义与用户可见行为的前提下，减少命令后缀解析与 GUI/网络入口重复，降低后续 triggerSource/core 语义演进成本。
