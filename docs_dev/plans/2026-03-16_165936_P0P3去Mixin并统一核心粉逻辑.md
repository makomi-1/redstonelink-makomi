# P0-P3 去 Mixin 并统一核心粉逻辑（保留锂兼容）

生成时间：2026-03-16 16:59:36  
文件名：2026-03-16_165936_P0P3去Mixin并统一核心粉逻辑.md

## 任务背景
当前分支需要抽离 mixin-dust 方案，要求顶面核心粉不再走特殊计算路径，去除顶面 ACTIVE 限流，并清理所有与 Mixin 注入直接相关的实现与配置。  
同时保留锂兼容能力的基础设施（如锂运行依赖与健康检查入口），不做“去锂”。

## 方案详情

### 现状分析
- `LinkRedstoneDustCoreBlock` 仍保留“顶面走原版 wire 计算、非顶面走内部同步”双路径。
- `LinkRedstoneDustCoreBlockEntity` 存在 `topActive*` 节流状态机。
- 工程仍注册 `redstonelink.mixins.json` 与 client mixin 配置，并保留 mixin 源码。
- 锂诊断中 `LithiumHitCounter` 依赖 mixin 命中计数，去 mixin 后会产生伪异常。

### 技术方案
1. 将核心粉状态更新逻辑统一到“非顶面路径”，移除顶面特殊计算入口。
2. 删除顶面 ACTIVE 客户端同步节流实现及对应配置项。
3. 删除所有 Mixin 代码与 Mixin 配置入口（包含主端与客户端示例）。
4. 保留锂兼容能力，但清理“依赖 mixin 命中”的诊断实现，避免无注入时误报。

### 影响范围
- 逻辑：`LinkRedstoneDustCoreBlock`、`LinkRedstoneDustCoreBlockEntity`
- 配置：`RedstoneLinkConfig`
- 资源配置：`fabric.mod.json`、`redstonelink.mixins.json`、`redstonelink.client.mixins.json`
- 代码目录：`com.makomi.mixin.*`、`com.makomi.mixin.client.*`
- 锂诊断：`LithiumCompatHealth`、`LithiumHitCounter`
- 脚本：`tools/ci/run_lithium_matrix.ps1`、`tools/ci/run_lithium_matrix.sh`

## 原子步骤清单

### 步骤 1：统一核心粉计算路径（P0）
- **操作对象**：`LinkRedstoneDustCoreBlock`
- **具体动作**：删除顶面特殊计算与暴露给 mixin 的静态桥接方法，统一使用内部 active->power 同步路径。
- **预期结果**：顶面与非顶面不再存在两套计算分支。
- **关键里程碑**：是

### 步骤 2：移除顶面 ACTIVE 节流（P1）
- **操作对象**：`LinkRedstoneDustCoreBlockEntity`、`RedstoneLinkConfig`
- **具体动作**：删除 `topActive*` 状态机、调度方法与配置项 `server.topActiveSyncThrottleTicks`。
- **预期结果**：ACTIVE 状态同步不再依赖节流窗口。
- **关键里程碑**：是

### 步骤 3：清理 Mixin 注入链路（P2）
- **操作对象**：mixin 源码与 mixin 配置文件
- **具体动作**：删除主端/客户端 mixin 类、plugin 与 json 配置，并从 `fabric.mod.json` 移除 mixins 声明。
- **预期结果**：运行时不再加载任何本模组 Mixin。
- **关键里程碑**：是

### 步骤 4：保留锂兼容并清理 mixin 关联诊断（P3）
- **操作对象**：`LithiumCompatHealth`、`LithiumHitCounter`、锂矩阵脚本
- **具体动作**：删除 `LithiumHitCounter` 及其引用，将锂诊断收敛为“动态方法签名检查”；移除脚本中的 mixin 调试参数。
- **预期结果**：锂兼容能力保留，且无 mixin 依赖误报。
- **关键里程碑**：是

### 步骤 5：回归验证
- **操作对象**：Gradle 任务链
- **具体动作**：执行 `.\gradlew.bat test testIntegration testClient testSlow testApiLegacy --no-daemon`。
- **预期结果**：核心构建与测试通过，变更可回归。
- **关键里程碑**：是
