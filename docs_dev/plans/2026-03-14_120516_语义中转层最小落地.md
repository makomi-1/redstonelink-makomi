# 语义中转层最小落地

生成时间：2026-03-14 12:05:16
文件名：2026-03-14_120516_语义中转层最小落地.md

## 任务背景
为修正底层实现导致的语义偏差，现有代码已经引入 `LinkNodeSemantics`，但命令层与配置层仍存在散点解析与重复校验。
本次先做“最小落地版本”：建立可复用语义内核并接入高价值入口，为新功能接入与老功能迁移提供统一中转层。

## 方案详情

### 现状分析
- 已有语义工具：`LinkNodeSemantics` 提供类型解析、角色校验、语义名称映射。
- 仍有重复解析：`ModCommands` 存在手写类型 `switch`，`CrossChunkCommandRegistry` 也有局部重复校验。
- 跨区块链路已使用语义约束，但入口层未完全统一，存在行为漂移风险。

### 技术方案
1. 新增 `com.makomi.data.semantic` 最小内核：
   - `SemanticTypeAliasResolver`：别名解析。
   - `SemanticRolePolicy`：角色校验与“按角色+配置允许集”统一校验。
   - `SemanticResult` + `SemanticError`：统一结果模型。
2. `LinkNodeSemantics` 保留兼容壳，内部委托新内核。
3. 首批迁移两处入口：
   - `CrossChunkCommandRegistry.parseType` 统一走语义中转校验。
   - `ModCommands.parseNodeTypeArg` 去除手写 `switch`。
4. 增加语义内核单测，验证别名解析、角色限制、配置限制。

### 影响范围
- `src/main/java/com/makomi/data/semantic/*`（新增）
- `src/main/java/com/makomi/data/LinkNodeSemantics.java`
- `src/main/java/com/makomi/command/crosschunk/CrossChunkCommandRegistry.java`
- `src/main/java/com/makomi/command/ModCommands.java`
- `src/test/java/com/makomi/data/semantic/*`（新增）
