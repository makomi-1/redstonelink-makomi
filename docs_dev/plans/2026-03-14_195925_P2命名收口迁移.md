## 任务背景
在 P0/P1 已完成“新增语义入口 + 调用点迁移”后，继续执行 P2：推进高风险命名收口，主要覆盖 GUI 与网络命名，确保行为不变且保留兼容壳。

## 方案详情

### 现状分析
- GUI 侧仍保留 `LinkPairingScreen` 历史命名，不符合 triggerSource 语义。
- 网络侧仍保留 `OpenButtonPairingPayload` 历史命名，不符合 triggerSource 语义。
- 调用点已具备按 sourceType 打开配对的统一能力，可安全承接命名迁移。

### 技术方案
- 新增 `TriggerSourcePairingScreen` 作为触发侧 GUI 主命名入口。
- `LinkPairingScreen` 保留为兼容壳并委托新类。
- 在 `PairingNetwork` 新增 `OpenTriggerSourcePairingPayload`，并以其作为注册与发送主入口。
- `OpenButtonPairingPayload` 保留为兼容壳（复用同一协议 ID 与编解码），避免行为变化。
- 客户端注册与测试统一切换到新命名入口。

### 影响范围
- `src/client/java/com/makomi/client/screen/TriggerSourcePairingScreen.java`
- `src/client/java/com/makomi/client/screen/LinkPairingScreen.java`
