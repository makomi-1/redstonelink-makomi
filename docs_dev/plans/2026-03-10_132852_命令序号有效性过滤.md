# 命令序号有效性过滤

生成时间：2026-03-10 13:28:52
文件名：2026-03-10_132852_命令序号有效性过滤.md

## 任务背景
命令行输入的序号缺少统一有效性过滤（已分配且未退役），希望复用配对 GUI 的序号过滤逻辑，降低重复实现。

## 方案详情

### 现状分析
- GUI 侧在 `AbstractMultiPairingScreen.parseTargets` 中做了序号解析与基础过滤。
- 命令侧在 `ModCommands.parseTargetSerials` 中存在重复实现，但未完全覆盖所有命令输入序号。
- `node get` / `link get` 允许输入无效序号作为诊断用途，保留不做过滤。

### 技术方案
- 提取通用序号解析工具类 `SerialParseUtil`（支持逗号/分号/空白分隔与纯数字校验）。
- GUI 与命令侧统一调用该工具，避免重复解析逻辑。
- 命令侧对需要输入序号的命令增加“已分配且未退役”校验，排除 `node get` / `link get`。

### 影响范围
- `src/main/java/com/makomi/util/SerialParseUtil.java`
- `src/client/java/com/makomi/client/screen/AbstractMultiPairingScreen.java`
- `src/main/java/com/makomi/command/ModCommands.java`
- `agents/plans/05_后续迭代与测试路线.md`
