# 锂兼容守护最小方案

生成时间：2026-03-08 13:21:05
文件名：2026-03-08_132105_锂兼容守护最小方案.md

## 任务背景
当前已完成 Lithium 兼容修复，但还缺少可观测的运行路由自检与可重复执行的有锂/无锂双矩阵校验，导致后续升级时容易出现静默失效。

## 方案详情

### 现状分析
- 已存在有锂/无锂二选一逻辑，但缺少启动期明确日志。
- 本地已有 L1 检查脚本，但没有专门覆盖 Lithium 开关矩阵。
- 动态注入使用 require=0，构建可通过但需要更强的运维可观测手段辅助排查。

### 技术方案
1. 在 `RedstoneLinkMixinPlugin` 增加一次性路由日志，打印 lithium 是否加载、两套 mixin 选择结果。
2. 新增 `tools/ci/run_lithium_matrix.ps1` 与 `tools/ci/run_lithium_matrix.sh`，默认执行：
   - 无锂：`compileJava + remapJar`
   - 有锂：`-PwithLithium=true compileJava + remapJar`
   并提供可选参数执行 `runClient` 烟测。
3. 更新 `agents/plans/05_后续迭代与测试路线.md` 追加本次守护步骤与命令。

### 影响范围
- `src/main/java/com/makomi/mixin/plugin/RedstoneLinkMixinPlugin.java`
- `tools/ci/run_lithium_matrix.ps1`
- `tools/ci/run_lithium_matrix.sh`
- `agents/plans/05_后续迭代与测试路线.md`
