# 启动期区块加载回放诊断隔离

生成时间：2026-03-20 17:48:14  
文件名：2026-03-20_174814_启动期区块加载回放诊断隔离.md

## 任务背景

旧存档进入世界时卡在 `Preparing spawn area: 0%`，且在关闭 `crosschunk.relay.enabled` 与 `crosschunk.forceLoad.enabled` 后问题仍存在。需要通过最小改动验证是否为“启动期区块批量加载触发的同步回放”阻塞主线程。

## 方案详情

### 现状分析

- `CHUNK_LOAD` 会触发 `publishChunkLifecycle(..., true)`。
- 该流程会遍历区块方块实体并展开链路发布 attach 事件，属于同步执行路径。
- 在启动准备区（spawn preparing）阶段，区块加载密集，可能导致同线程压力过高。

### 技术方案

- 在 `LinkNodeLifecycleDispatchEvents` 中增加“启动期诊断隔离门”：
  - 仅当 `online=true` 且 `gameTime<=0`（启动最早期）时，跳过 attach 回放。
  - 增加限流日志，确认是否命中隔离门。
- 不改动 `CHUNK_UNLOAD`（detach/remove）路径。
- 该变更用于定位问题来源，后续可根据结果决定改为延后消费/分帧预算等正式方案。

### 影响范围

- `src/main/java/com/makomi/data/LinkNodeLifecycleDispatchEvents.java`

## 原子步骤清单

- **操作对象**：`LinkNodeLifecycleDispatchEvents`
- **具体动作**：添加启动期判定与日志限流，命中时直接返回，不执行 `publishLinkAttached`
- **预期结果**：启动准备阶段不再执行大批量 attach 回放，便于确认卡顿来源
