# 统一退役入口并强同步白名单

生成时间：2026-03-15 14:26:36
文件名：2026-03-15_142636_统一退役入口并强同步白名单.md

## 任务背景
当前退役调用分散在命令、批量命令、事件与方块实体等多个位置，且白名单清理属于独立动作。  
为降低遗漏风险，需要将“退役 + 白名单强同步清理”收口为一个内部统一入口。

## 方案详情

### 现状分析
- 多个位置直接调用 `LinkSavedData.retireNode(...)`。
- 白名单清理逻辑未与退役强绑定，存在一致性风险。

### 技术方案
- 新增内部协调类，提供统一方法 `retireAndSyncWhitelist(level, type, serial)`。
- 统一方法内部执行顺序：
  1. 调用 `retireNode` 完成退役；
  2. 强同步清理白名单（含 resident）中该 `type+serial` 的 source/target 条目。
- 将所有直接 `retireNode` 调用改为统一入口调用。

- 不改变现有命令语义与调度边界，仅提升一致性与可维护性。
