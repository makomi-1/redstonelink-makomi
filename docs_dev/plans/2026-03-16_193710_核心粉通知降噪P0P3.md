# 核心粉通知降噪 P0-P3

生成时间：2026-03-16 19:37:10
文件名：2026-03-16_193710_核心粉通知降噪P0P3.md

## 任务背景
当前顶面与非顶面核心粉在激活态切换路径上已统一，但邻居通知存在重复扇出，导致单次状态翻转通知量偏高。目标是在不破坏“直接附着/隔块生效”前提下减少通知量。

## 方案详情

### 现状分析
- `LinkRedstoneDustCoreBlockEntity.onActiveChanged` 在委托方块侧同步后，仍执行一次“中心 + 六方向”手动通知。
- `LinkRedstoneDustCoreBlock.notifyAttachedNeighbor` 也执行“附着块 + 六方向”通知。
- `syncPowerWithActiveState` 仍通过 `setBlock(..., UPDATE_ALL)` 触发基础邻居更新。

### 技术方案
- 采用分步降噪，先删明显重复通知，再收敛附着块扇出，最后通过全量测试回归验证。
- 暂不触及 `setBlock` flags，避免把问题耦合到更新语义层。

### 影响范围
- `src/main/java/com/makomi/block/entity/LinkRedstoneDustCoreBlockEntity.java`
- `src/main/java/com/makomi/block/LinkRedstoneDustCoreBlock.java`
- `src/test/java/com/makomi/block/LinkRedstoneDustCoreStateContractTest.java`

## 原子步骤清单

### 步骤 1：删除 BlockEntity 侧重复扇出
- **操作对象**：`LinkRedstoneDustCoreBlockEntity.onActiveChanged`
- **具体动作**：移除委托后对 `worldPosition` 的“中心 + 六方向”手动 `updateNeighborsAt`。
- **预期结果**：保留方块侧通知主路径，减少一整层重复通知。
- **关键里程碑**：是

### 步骤 2：收敛附着块通知
- **操作对象**：`LinkRedstoneDustCoreBlock.notifyAttachedNeighbor`
- **具体动作**：由“附着块 + 六方向”改为仅 `updateNeighborsAt(attachedPos, block)`。
- **预期结果**：在维持隔块链路可达的前提下减少二次扇出。
- **关键里程碑**：是

### 步骤 3：补充契约约束
- **操作对象**：`LinkRedstoneDustCoreStateContractTest`
- **具体动作**：增加/调整源码契约断言，防止重复扇出回归。
- **预期结果**：后续修改可自动感知通知量回退。
- **关键里程碑**：否

### 步骤 4：全量回归
- **操作对象**：Gradle 测试任务
- **具体动作**：执行 `.\gradlew.bat test testIntegration testClient testSlow testApiLegacy --no-daemon`。
- **预期结果**：通知降噪改动不引入行为回归。
- **关键里程碑**：是

## 预期结果
核心粉状态翻转通知量显著下降，顶面与非顶面保持同一路径语义，并通过自动化回归验证稳定性。
