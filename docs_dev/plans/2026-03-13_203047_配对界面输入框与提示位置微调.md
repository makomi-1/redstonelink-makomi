# 配对界面输入框与提示位置微调

## 任务背景
用户反馈配对界面存在两个体验问题：
1) 输入框在部分场景会“跳动”，示例提示不易看到；
2) 非法输入提示与按钮区域位置冲突，需要移动到按钮下方且保持靠近。

## 方案详情

### 现状分析
- 输入框初始化时固定写入 `joinTargets(currentTargets)`，导致非空时光标位置偏向末尾。
- 非法提示使用固定坐标绘制，未基于按钮位置进行锚定计算。

### 技术方案
- 输入框初始化改为条件写入：仅在初始化文本非空时设置输入值；为空时保持默认空串，让示例提示可见。
- 非法提示位置改为按钮行底部 + 固定间距，通过按钮基准坐标计算而非硬编码常量。

### 影响范围
- `src/client/java/com/makomi/client/screen/AbstractMultiPairingScreen.java`

## 原子步骤清单

### 步骤 1：输入框初始化条件化
- **操作对象**：`AbstractMultiPairingScreen`
- **具体动作**：新增初始化文本变量，仅当非空时调用 `serialInput.setValue(...)`。
- **预期结果**：空输入不跳动且示例提示可见，非空场景保持原行为。
- **关键里程碑**：是

### 步骤 2：错误提示位置重算
- **操作对象**：`AbstractMultiPairingScreen`
- **具体动作**：以按钮行位置为锚点，使用“按钮底部 + 边距”绘制非法提示。
- **预期结果**：提示位于按钮下方、靠近且不重合。
- **关键里程碑**：是

### 步骤 3：编译验证
- **操作对象**：Gradle 编译任务

## 预期结果

- 非法提示在按钮下方近距离显示，避免重合。
