# bench suite 每 case 新档编排

生成时间：2026-03-22 19:17:08

## 任务背景
当前 `tools/bench/run-bench.ps1` 只负责“在一个已指定世界里跑一个 case”。如果要做到“每个 case 一个全新世界目录，且 dedicated server 重启后只加载这个世界”，关键不在单 case 脚本内部，而在 dedicated server 生命周期外层增加一层 suite 编排。

## 方案详情

### 现状分析
- 单 case bench 已具备 datapack 安装、自动放置、建链、驱动、spark 结果写回与结果 JSON 落盘能力。
- 现有脚本不会管理外部 dedicated server 进程，也不会切换 `server.properties` 的 `level-name`。
- 因为 Minecraft dedicated server 单次进程只会加载一个 `level-name`，所以“一个 case 一个新档”必须通过“复制模板世界 -> 改 level-name -> 启服 -> 跑 case -> 停服”的外层调度完成。

### 技术方案
- 新增 `tools/bench/run-bench-suite.ps1`，作为 suite/orchestrator 脚本。
- suite 脚本只负责：
  1. 读取 matrix，解析 case 列表。
  2. 将模板世界复制为 `rl-case-<caseId>-<timestamp>` 新目录。
  3. 最小修改 dedicated server 的 `server.properties`，仅切换 `level-name`。
  4. 启动外部 dedicated server，并轮询 RCON 就绪。
  5. 直接调用现有 `run-bench.ps1 -Action RunCase` 运行单 case。
  6. 从 bench 结果目录中识别本次新增的结果 JSON，并回填 spark URL/世界路径等摘要。
  7. 通过 RCON 发送 `stop`，等待服务端进程退出。
  8. 全部 case 完成后恢复原始 `server.properties` 内容，并落盘 suite summary。
- 默认保留 case 世界目录，不自动删除；仅在显式传入删除开关时才清理成功 case 的世界。
- 默认失败即停；若用户明确需要全量跑完，可通过继续执行开关允许后续 case 继续。
