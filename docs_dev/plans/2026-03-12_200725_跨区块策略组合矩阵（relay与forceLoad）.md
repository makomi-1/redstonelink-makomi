# 跨区块策略组合矩阵（relay与forceLoad）

生成时间：2026-03-12 20:07:25  
文件名：2026-03-12_200725_跨区块策略组合矩阵（relay与forceLoad）.md

## 任务背景
- 已实现跨区块调度、白名单与强制加载限流，但当前缺少“中继缓冲独立开关”与“强制加载模式切换”。
- 需求是新增 `crosschunk.forceLoad.mode=all/whitelist`，并与 `relay` 开关做组合使用，形成最终判定矩阵。

## 方案详情

### 现状分析
- 未加载目标时默认都会进入中继调度队列。
- 强制加载目前只支持“命中白名单/preset”语义，缺少全量模式。
- 没有显式矩阵表达“relay 与 forceLoad 的交叉组合”。

### 技术方案
- 在配置层新增：
  - `crosschunk.relay.enabled`（默认 `true`）
  - `crosschunk.forceLoad.mode`（`all|whitelist`，默认 `whitelist`）
- 在调度层新增“可入队判定”：
  - `allowQueue = relayEnabled || canForceLoad`
  - `relay=false` 且 `canForceLoad=false` 时直接丢弃未加载目标触发
- 在强制加载判定中新增模式分支：
  - `mode=all`：类型允许即强制加载（不看白名单）
  - `mode=whitelist`：维持现有白名单/preset 逻辑

- `src/main/java/com/makomi/data/CrossChunkDispatchService.java`
