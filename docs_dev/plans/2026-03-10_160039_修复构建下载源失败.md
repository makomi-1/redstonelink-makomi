# 修复构建下载源失败

生成时间：2026-03-10 16:00:39
文件名：2026-03-10_160039_修复构建下载源失败.md

## 任务背景
在执行 ./gradlew.bat build 时，构建在 Loom 配置阶段失败。日志显示下载 Minecraft 版本清单时访问 bmcl 镜像失败，导致项目无法继续构建。

## 方案详情

### 现状分析
- 当前 gradle.properties 显式覆盖了 Loom 资源与版本清单地址。
- 覆盖地址指向 https://bmclapi2.bangbang93.com。
- 在当前网络环境下该地址连接失败（ConnectException），触发 Failed download after 3 attempts。

### 技术方案
- 保持配置结构不变，采用最小改动方案：将 Loom 覆盖地址替换为官方地址。
- 不调整 build.gradle 与 settings.gradle 逻辑，避免引入额外副作用。

### 影响范围
- gradle.properties 的 Loom 下载源配置。
- 构建流程中的依赖与版本清单下载阶段。

## 原子步骤清单

### 步骤 1：替换 Loom 下载覆盖地址
- **操作对象**：gradle.properties
- **具体动作**：将 bmcl 的资源与版本清单 URL 替换为官方 URL
- **预期结果**：Loom 下载阶段不再命中不可达镜像
- **关键里程碑**：是

### 步骤 2：执行构建验证
- **操作对象**：Gradle 构建流程
- **具体动作**：运行 ./gradlew.bat build --stacktrace
- **预期结果**：通过下载阶段并完成构建，或暴露下一层问题
- **关键里程碑**：是

## 预期结果
- 构建恢复可执行状态。
- 避免因单一不可达镜像导致的构建中断。
