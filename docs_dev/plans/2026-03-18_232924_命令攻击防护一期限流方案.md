# 命令攻击防护一期限流方案

生成时间：2026-03-18 23:29:24
文件名：2026-03-18_232924_命令攻击防护一期限流方案.md

## 任务背景

当前命令体系已有权限门槛与输入上限，但缺少“按来源身份隔离”的频率防护。  
仅按权限等级限流会导致同权限玩家共享容量，存在单人独占风险。  
本次目标是在不改变现有命令语义的前提下，为命令入口增加差异化频率防护。

## 方案详情

### 现状分析

- 现有命令保护主要是 `requires(permission)` 与批量上限、输入长度上限。
- 尚未存在统一命令限流服务，也没有“玩家身份维度”频控。
- 高频风险主要集中在 `link` 写入与 `crosschunk` 管理命令。

### 技术方案

- 新增统一限流服务 `CommandRateLimitService`，采用基于 `gameTime` 的窗口计数器。
- 限流判定采用四层闸门：
  1. 全局容量（Global）
  2. 权限层级容量（Tier）
  3. 来源个体容量（Actor）
  4. 来源+命令组容量（ActorGroup）
- 来源个体键使用多信息位：来源类型 + 主体标识 + 权限等级，避免同权限玩家抢占同一桶。
- 一期命令组接入范围：
  - `LINK_RW`：`link add/remove/set` 与 `link write_control protected *`
  - `CROSSCHUNK`：`crosschunk` 命令组
- 拒绝提示统一为“没有足够权限/系统细节不外露”风格中的通用频率提示，不回显阈值。

## 原子步骤清单

### 步骤 1：新增限流服务骨架
- **操作对象**：`CommandRateLimitService`
- **具体动作**：实现分组枚举、来源身份键构造、窗口计数器与统一判定入口
- **预期结果**：提供 `tryAcquire(CommandSourceStack, group, cost)` 统一能力
- **关键里程碑**：是

### 步骤 2：扩展服务端配置项
- **操作对象**：`RedstoneLinkConfig`
- **具体动作**：新增 rateLimit 配置项读取、默认值与 getter
- **预期结果**：限流参数可由服务端配置统一裁决
- **关键里程碑**：是

### 步骤 3：接入 link 命令入口
- **操作对象**：`ModCommands`
- **具体动作**：在 `link` 写路径入口调用限流服务，并按输入规模给出成本
- **预期结果**：高频写入命令被分层限流
- **关键里程碑**：是

### 步骤 4：接入 crosschunk 命令入口
- **操作对象**：`CrossChunkCommandRegistry`
- **具体动作**：在 `crosschunk` 执行入口调用限流服务，并对 set 命令使用批量成本
- **预期结果**：跨区块管理命令具备抗刷能力
- **关键里程碑**：是

### 步骤 5：文案与说明更新
- **操作对象**：语言文件与 `docs_dev/使用说明.md`
- **具体动作**：新增超限提示键值，补充配置项与推荐用法说明
- **预期结果**：提示与行为一致，便于服主管理
- **关键里程碑**：否

### 步骤 6：验证与回归
- **操作对象**：Gradle 构建与测试
- **具体动作**：执行编译和全标签测试门禁
- **预期结果**：改动无编译错误且回归通过
- **关键里程碑**：是

## 预期结果

在不改变命令业务语义的前提下，实现“按权限分档 + 按玩家身份隔离 + 按命令组细分”的命令频控。  
一期覆盖 `link` 与 `crosschunk` 高频面，显著降低命令刷频攻击影响。
