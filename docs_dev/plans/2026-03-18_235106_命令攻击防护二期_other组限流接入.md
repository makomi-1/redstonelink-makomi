# 命令攻击防护二期：other 组限流接入

生成时间：2026-03-18 23:51:06
文件名：2026-03-18_235106_命令攻击防护二期_other组限流接入.md

## 任务背景

一期已完成 `link` 与 `crosschunk` 命令组分层限流。  
当前仍有“其他权限”命令组（activate/retire/place/audit/node-get-list/link-get）未接入统一限流，存在高频调用攻击面。

## 方案详情

### 现状分析

- 命令分组在权限层面已存在：`server.command.otherPermissionLevel`。
- 限流服务已支持多层闸门，但分组仅覆盖 `LINK_RW` 与 `CROSSCHUNK`。
- `other` 组中包含批量与潜在高成本命令（尤其 `place fill`、`activate batch`、`retire batch`）。

### 技术方案

- 在 `CommandRateLimitService.CommandGroup` 中新增 `OTHER`。
- 在配置层新增 `server.command.rateLimit.actorGroup.other.baseCapacity/stepPerLevel`。
- 将以下命令入口接入 `OTHER` 组限流：
  1. `node activate`（批量成本）
  2. `node retire`（单条确认与执行）
  3. `node retire batch`（批量成本）
  4. `node get/list`、`link get`、`audit`（低成本）
  5. `place setblock/fill/confirm`（按操作规模给成本）
- 保持统一失败提示，不暴露阈值细节。
