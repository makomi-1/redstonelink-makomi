# 跨区块触发提示

生成时间：2026-03-13 14:17:45
文件名：2026-03-13_141745_跨区块触发提示.md

## 任务背景
跨区块派发目前只返回是否处理成功，缺少“强加载/中继缓冲接管”的提示与可读性结果。需要在触发端提供跨区块接管生效提示，并支持 simple/detailed 模式与开关配置，同时保证不改变既有派发语义。

## 方案详情

### 现状分析
- LinkedTargetDispatchService 调用 CrossChunkDispatchService.queue* 时只拿到布尔结果，无法区分强加载/中继缓冲。
- 触发入口集中在 TriggerSourceBlockEntity 与 LinkerItem，适合作为统一提示出口。
- 配置中暂无跨区块提示的开关与模式字段。

### 技术方案
- 仅扩展返回结构：queueActivation/queueSyncSignal 返回 QueueResult(accepted, forceLoadPlanned)，不改现有派发语义。
- 在 LinkedTargetDispatchService 汇总跨区块接管结果，区分强加载/中继缓冲的目标列表。
- 在触发端提示：仅当“跨区块接管生效”时提示；simple/detailed 模式仅影响条数上限（3/50），超出显示 (+n)；不做同 tick 合并。
- 增加配置项 crosschunk.notify.enabled 与 crosschunk.notify.mode（默认 simple），并提供语言包文案与使用说明。

### 影响范围
- 代码：CrossChunkDispatchService、LinkedTargetDispatchService、TriggerSourceBlockEntity、LinkerItem、RedstoneLinkConfig
- 资源：zh_cn.json、en_us.json
- 文档：docs_dev/使用说明.md
