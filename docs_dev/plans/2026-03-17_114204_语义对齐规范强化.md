# 语义对齐规范强化

生成时间：2026-03-17 11:42:04
文件名：2026-03-17_114204_语义对齐规范强化.md

## 任务背景
用户要求强化 `AGENTS.md` 与 `skills` 规则，确保后续新设计实现严格对齐现有语义设计，避免 triggerSource/core 语义漂移与命名混用。

## 方案详情

### 现状分析
- `AGENTS.md` 已有语义规范，但约束粒度偏粗，缺少实施前检查、兼容边界与交付证据要求。
- `skills/docs-dev-docflow/SKILL.md` 对语义对齐的约束较弱，尚未将语义检查固化为 plans 必填字段。

### 技术方案
- 在 `AGENTS.md` 增补“语义对齐强约束”：
  - 术语基线（triggerSource/core 与 `LinkNodeType` 映射）
  - 语义方向隔离（triggerSource 仅来源，core 仅目标）
  - 实施前语义检查清单（命名、I/O、文案、映射、兼容）
  - 兼容边界（旧别名仅输入兼容，输出统一新术语）
  - 交付证据要求（方案确认/任务汇报需给出语义对齐说明）
- 在 `skills/docs-dev-docflow/SKILL.md` 同步强化：
  - 总体原则加入语义术语与兼容边界规则
  - `docs_dev/plans` 必填字段新增“语义对齐检查结果”
  - 输出要求加入“若命中语义变更，汇报语义对齐结论”

### 影响范围
- 仅影响规范文档：`AGENTS.md`、`skills/docs-dev-docflow/SKILL.md`
- 不影响业务逻辑与运行时行为。

- **预期结果**：文档工作流与 AGENTS 规则一致
- **关键里程碑**：是
