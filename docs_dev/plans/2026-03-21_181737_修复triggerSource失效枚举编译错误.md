# 修复 triggerSource 失效枚举编译错误

生成时间：2026-03-21 18:17:37
文件名：2026-03-21_181737_修复triggerSource失效枚举编译错误.md

## 任务背景
当前编译失败集中在 `triggerSource` 失效语义拆分后的枚举不一致：
- 调用方已使用 `TRIGGER_SOURCE_CHUNK_UNLOAD_INVALIDATION` 与 `TRIGGER_SOURCE_INVALIDATION`
- `ActivatableTargetBlockEntity.DeltaKind` 仍停留在旧的 `SOURCE_INVALIDATION`

结果是 `compileJava` 在多个调用点报“找不到符号”。

## 方案详情

### 现状分析
- `CrossChunkDispatchService`、`InternalDispatchDeltaEvents`、`InternalDispatchDeltaProjector` 已按新语义引用新枚举值。
- 目标侧 `applyDispatchDelta(...)` 仍只识别旧的 `SOURCE_INVALIDATION`。
- 目标侧失效处理方法也还未拆成“区块卸载 soft invalidation”和“其它失效 hard invalidation”。

### 技术方案
- 在 `ActivatableTargetBlockEntity.DeltaKind` 中补齐：
  - `TRIGGER_SOURCE_CHUNK_UNLOAD_INVALIDATION`
  - `TRIGGER_SOURCE_INVALIDATION`
- 保留 `SOURCE_INVALIDATION` 作为 legacy alias。
- 将目标侧分发逻辑改为：
  - `SOURCE_INVALIDATION` / `TRIGGER_SOURCE_CHUNK_UNLOAD_INVALIDATION` -> 仅清 `sync`
  - `TRIGGER_SOURCE_INVALIDATION` -> 清 `toggle/pulse/sync`

### 影响范围
- `src/main/java/com/makomi/block/entity/ActivatableTargetBlockEntity.java`
- 编译链路：`compileJava`、`compileClientJava`、`compileTestJava`

## 原子步骤清单

### 步骤 1：补齐 DeltaKind 枚举
- **操作对象**：`ActivatableTargetBlockEntity.DeltaKind`
- **具体动作**：加入两个新失效枚举值，保留旧枚举值用于兼容
- **预期结果**：外部调用方引用的枚举值可被解析
- **关键里程碑**：是

### 步骤 2：补齐目标侧失效分发
- **操作对象**：`ActivatableTargetBlockEntity.applyDispatchDelta(...)`
- **具体动作**：按 soft/hard 两类失效分流到不同私有处理方法
- **预期结果**：目标侧实现与新语义一致
- **关键里程碑**：是

### 步骤 3：恢复 hard invalidation 处理逻辑
- **操作对象**：`ActivatableTargetBlockEntity`
- **具体动作**：新增“其它失效”处理方法，剔除 `toggle/pulse/sync` 贡献并重算
- **预期结果**：`triggerSource` 其它失效语义可正常编译和执行
- **关键里程碑**：是

### 步骤 4：编译验证
- **操作对象**：Gradle 编译任务
- **具体动作**：执行 `compileJava compileClientJava compileTestJava`
- **预期结果**：当前编译错误消失，若有后续错误继续收敛
- **关键里程碑**：是
