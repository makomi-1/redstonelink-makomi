# fill dry_run 移除

生成时间：2026-03-21 14:59:55

## 任务背景
- 现有 `place fill` 仍保留 `dry_run` 子命令。
- 目标是移除 `fill dry_run`，保留 `fill` 的 `force/confirm` 流程。

## 方案详情

### 现状分析
- `place fill` 命令树包含 `dry_run` 分支。
- `ModCommands` 存在 `executePlaceFillDryRun` 与 `executePlaceFillInternal` 的 `dryRun` 分支。
- 多语言资源中存在 `message.redstonelink.place.dry_run_fill` 文案。

### 技术方案
- 删除命令树中的 `place fill ... dry_run` 入口。
- 删除 `executePlaceFillDryRun` 方法与 `fill` 内部 `dryRun` 执行分支。
- 清理 `dry_run_fill` 文案，并更新使用说明描述。

### 影响范围
- 命令实现：`ModCommands`
- 文案资源：`zh_cn.json`、`en_us.json`
- 使用说明：`docs_dev/使用说明.md`
