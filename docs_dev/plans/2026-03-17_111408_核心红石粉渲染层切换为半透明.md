# 核心红石粉渲染层切换为半透明

生成时间：2026-03-17 11:14:08
文件名：2026-03-17_111408_核心红石粉渲染层切换为半透明.md

## 任务背景
“连接红石粉核心”在物品栏与手持场景未体现半透明效果。已确认并非贴图与资源覆盖问题，核心差异在于该方块当前客户端渲染层使用 `cutout`，而透明变种使用 `translucent`。

## 方案详情

### 现状分析
- 普通核心红石粉：`BlockRenderLayerMap` 映射为 `RenderType.cutout()`。
- 透明变种核心红石粉：映射为 `RenderType.translucent()`。
- 普通核心红石粉物品为 `BlockItem`，其手持/物品栏表现受方块渲染层路径影响，单改 item model 的 `render_type` 不足以覆盖该差异。

### 技术方案
- 仅修改客户端渲染层注册：
  - 将 `ModBlocks.LINK_REDSTONE_DUST_CORE` 从 `RenderType.cutout()` 改为 `RenderType.translucent()`。
- 维持其它渲染层映射与资源文件不变。

### 影响范围
- 影响：核心红石粉的物品栏、手持与放置渲染均切换到半透明混合路径。
- 不影响：序号外显逻辑、命令逻辑、贴图资源文件内容。
