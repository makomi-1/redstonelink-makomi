## 任务背景

## 方案详情

### 现状分析
- 已有 stable-core 与 integration 的基础用例，但缺少多样本的 NBT 兼容回归。
- slow 标签未形成稳定的高负载回归用例。
- 集成层流程覆盖仍偏少。

### 技术方案
1. 新增历史/异常 SNBT fixture，覆盖缺字段、异常维度与空链接。
2. 在兼容性测试中引入参数化读取与断言，确保兼容行为可持续回归。
3. 新增 slow 回归用例，覆盖大规模序列与链接的保存/加载。
4. 扩展集成流程用例，覆盖“建立→解除→再绑定”关键链路。
5. 新增用例均遵循标签分层，不引入 API 相关测试。

### 影响范围
- `src/test/resources/fixtures/nbt/**`
