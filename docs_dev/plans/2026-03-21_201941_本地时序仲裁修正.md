# 本地时序仲裁修正

生成时间：2026-03-21 20:19:41
文件名：2026-03-21_201941_本地时序仲裁修正.md

## 任务背景
当前目标端在非跨区块路径下仍会保留旧时间粒度的 `toggle` 结构真值，导致更晚的 `sync/pulse` 结束后，旧 `toggle` 还能重新接管输出，和“时间粒度外按时序优先”的预期语义不一致。

## 方案详情

### 现状分析
- 目标端按 `sync/pulse/toggle` 三类并发桶保存结构真值。
- 现有实现会从全部历史桶中重算候选，旧 `toggle` 会在更晚事件结束后复活。
- `pulse` 活跃窗口内，如果来了更晚 `toggle`，当前实现也可能直接被更晚 `toggle` 抢占，不符合“toggle 不能阻止 pulse 下落”的预期。

### 技术方案
- 当更晚时间粒度到来时，淘汰更早帧：
  - 新 `sync`：淘汰更早的 `sync/toggle`，并取消 `pulse` 武装位与脉冲桶。
  - 新 `pulse`：淘汰更早的 `sync/toggle/pulse`。
  - 新 `toggle`：淘汰更早的 `sync/toggle`，但不打断仍在生效窗口中的 `pulse`。
- `pulse` 活跃窗口内，候选选择改为：
  - `sync` 与 `pulse` 仍按时间粒度和同粒度优先级裁决。
  - 若当前赢家是 `pulse`，则忽略 `toggle` 候选，直到 `pulse` 结束。
- 每次 `sync/pulse` 真值变化后，都同步重算 `toggle` 真值，避免基准态变化后 `toggleState` 滞留旧值。

### 影响范围
- `src/main/java/com/makomi/block/entity/ActivatableTargetBlockEntity.java`
- `src/test/java/com/makomi/block/entity/ActivatableTargetBlockEntityInternalTest.java`

## 预期结果
- 非跨区块路径下，时间粒度外按时序优先。
- 旧 `toggle` 不再在更晚 `sync/pulse` 结束后复活。
- later `toggle` 在 pulse 下落窗口内仅作为后续候选，不打断当前 pulse。
