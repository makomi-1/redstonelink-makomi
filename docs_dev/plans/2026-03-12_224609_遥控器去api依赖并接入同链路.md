# 遥控器去api依赖并接入同链路

生成时间：2026-03-12 22:46:09
文件名：2026-03-12_224609_遥控器去api依赖并接入同链路.md

## 任务背景
当前两个遥控器（toggle/pulse）仍依赖 com.makomi.api.v1 触发接口，导致其触发链路与按钮/拉杆分离，跨区块行为与维护路径不一致。目标是在不改变用户交互方式的前提下，移除业务层 API 依赖并尽量复用现有触发实现。

## 方案详情

### 现状分析
- LinkerItem 当前通过 RedstoneLinkApi.trigger().emit 发起触发。
- TriggerSourceBlockEntity 已具备跨区块调度接入（relay/forceLoad）与本地触发分支。
- ModItems 仍使用 ApiActivationMode 构造遥控器实例。

### 技术方案
- 新增内部派发复用服务，沉淀“来源序号 -> 目标序号集合”的激活/同步派发逻辑。
- TriggerSourceBlockEntity 改为调用复用服务，保持原语义与消息行为。
- LinkerItem 移除 com.makomi.api.v1 依赖，改用内部 ActivationMode 与复用服务派发。
- ModItems 改用内部 ActivationMode 构造两个遥控器。
- 使用说明补充“遥控器已与触发源同链路”的行为说明。

### 影响范围
- src/main/java/com/makomi/item/LinkerItem.java
- src/main/java/com/makomi/registry/ModItems.java
- src/main/java/com/makomi/block/entity/TriggerSourceBlockEntity.java
- src/main/java/com/makomi/data/（新增复用派发类）
- docs_dev/使用说明.md

## 原子步骤清单

### 步骤 1：新增复用派发服务
- **操作对象**：src/main/java/com/makomi/data/
- **具体动作**：抽取激活/同步派发公共逻辑，返回统一统计结果。
- **预期结果**：方块触发与遥控器可复用同一派发链路。
- **关键里程碑**：是

### 步骤 2：改造 TriggerSourceBlockEntity
- **操作对象**：src/main/java/com/makomi/block/entity/TriggerSourceBlockEntity.java
- **具体动作**：改为调用复用服务替代内联循环派发。
- **预期结果**：触发源逻辑不变且代码去重。
- **关键里程碑**：是

### 步骤 3：改造 LinkerItem 与 ModItems
- **操作对象**：src/main/java/com/makomi/item/LinkerItem.java、src/main/java/com/makomi/registry/ModItems.java
- **具体动作**：去除 API 依赖并接入内部模式枚举/派发逻辑。
- **预期结果**：两个遥控器去 API 依赖并进入统一链路。
- **关键里程碑**：是

### 步骤 4：同步使用说明
- **操作对象**：docs_dev/使用说明.md
- **具体动作**：在正文前部补充遥控器触发链路说明。
- **预期结果**：用户可感知行为变更有据可查。
- **关键里程碑**：否

## 预期结果
- 遥控器业务路径不再依赖 com.makomi.api.v1。
- 按钮/拉杆/遥控器的触发分发逻辑统一，跨区块行为一致性提升。
- 在不改变命令与交互入口的前提下，降低后续功能演进的维护成本。
