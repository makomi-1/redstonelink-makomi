# 写入控制模式与受控名单落地

生成时间：2026-03-18 10:09:18  
文件名：2026-03-18_100918_写入控制模式与受控名单落地.md

## 任务背景

当前链接写入路径已统一到命令入口与增量替换底层，但尚未具备“全量写入/限量写入/只读 + 受控名单”的统一写入控制。  
本次目标是在不改变 triggerSource/core 语义与方向约束前提下，为最终写入路径增加统一门禁。

## 方案详情

### 现状分析

- 最终链接写入入口集中在 `ModCommands` 的 `executeLinkUpdate/executeLinkSet`。  
- GUI 配对最终发送 `redstonelink link set ...`，本质仍走服务端命令入口。  
- 批量覆盖已具备增量底层，不需要在工具层增加权限语义。

### 技术方案

1. 扩展服务端配置：新增 `server.linkWriteControl.*` 配置组（mode、limited 权限与最大设置量、protected 查看与管理权限）。  
2. 新增受控名单持久化：`LinkWriteProtectedSavedData`，按 `type+serial` 管理。  
3. 新增统一判定服务：`LinkWriteControlService`，在最终写入前判定：
   - `readonly`：拒绝写入  
   - `limited`：按“最多设置量”限制（`setSize`）  
   - 受控名单：命中受控节点且权限不足则拒绝
4. 新增管理命令：`/redstonelink write_control protected add/remove/set/list`。  
5. 接入最终写入口：`executeLinkUpdate/executeLinkSet` 在落库前统一调用写控服务。

### 影响范围

- `RedstoneLinkConfig`（配置项与解析）  
- `ModCommands`（写入门禁与受控名单命令）  
- 新增 `LinkWriteProtectedSavedData`、`LinkWriteControlService`  
- `zh_cn/en_us` 文案

## 原子步骤清单

### 步骤 1：配置扩展
- **操作对象**：`RedstoneLinkConfig`
- **具体动作**：新增 linkWriteControl 配置读取、默认值、getter 与默认配置注释。
- **预期结果**：写控模式与阈值可配置。

### 步骤 2：受控名单持久化
- **操作对象**：`LinkWriteProtectedSavedData`
- **具体动作**：实现 `contains/add/remove/replace/list`。
- **预期结果**：受控名单可持久化与查询。

### 步骤 3：统一写控服务
- **操作对象**：`LinkWriteControlService`
- **具体动作**：封装 readonly/limited/protected 判定并返回拒绝原因。
- **预期结果**：写入控制逻辑单点维护。

### 步骤 4：命令接入
- **操作对象**：`ModCommands`
- **具体动作**：新增 `write_control protected` 子命令，并在链接写入入口接入写控判定。
- **预期结果**：所有链接写入统一受控。

### 步骤 5：文案与验证
- **操作对象**：语言文件与回归门禁
- **具体动作**：补充失败/管理文案，执行编译与全测试。
- **预期结果**：行为可观察、无回归。

## 预期结果

- 新增写入控制模式：`full/limited/readonly`。  
- `limited` 基于“最多设置量”而非增量差异量判定。  
- 受控名单在最终写入前生效，权限不足拒绝写入。  
- 现有 triggerSource/core 语义、方向约束、增量工具职责保持不变。
