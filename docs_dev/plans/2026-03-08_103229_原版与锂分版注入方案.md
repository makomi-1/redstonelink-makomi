# 原版与锂分版注入方案

生成时间：2026-03-08 10:32:29
文件名：2026-03-08_103229_原版与锂分版注入方案.md

## 任务背景
用户要求将红石注入拆为“原版路径”和“lithium 兼容路径”两套，避免无锂环境承载锂私有方法注入，并控制兼容逻辑启停边界。

## 方案详情

### 现状分析
- 现有 `RedStoneWireBlockMixin` 同时承载原版与 lithium 注入。
- lithium 相关注入目标包含运行时注入方法（`getReceivedPower/getPowerFromSide/getStrongPowerTo`），会在 remap 阶段产生提示。

### 技术方案
- 保留 `RedStoneWireBlockMixin` 作为原版链路注入入口。
- 新增 `RedStoneWireBlockLithiumMixin` 存放 lithium 专用注入。
- 新增 `RedstoneLinkMixinPlugin`，在 `shouldApplyMixin` 中通过 `FabricLoader.isModLoaded("lithium")` 控制是否启用 lithium mixin。
- 在 `redstonelink.mixins.json` 注册 plugin 和两类 mixin。

### 影响范围
- `src/main/java/com/makomi/mixin/RedStoneWireBlockMixin.java`
- `src/main/java/com/makomi/mixin/RedStoneWireBlockLithiumMixin.java`
- `src/main/java/com/makomi/mixin/plugin/RedstoneLinkMixinPlugin.java`
- `src/main/resources/redstonelink.mixins.json`

## 原子步骤清单

### 步骤 1：拆分 mixin
- **操作对象**：`RedStoneWireBlockMixin` 与新建 `RedStoneWireBlockLithiumMixin`
- **具体动作**：将 lithium 专用注入从原文件迁出并集中到新文件
- **预期结果**：原版注入与锂兼容注入边界清晰
- **关键里程碑**：是

### 步骤 2：接入条件启用
- **操作对象**：新建 `RedstoneLinkMixinPlugin` 与 `redstonelink.mixins.json`
- **具体动作**：按是否加载 lithium 控制 lithium mixin 是否应用
- **预期结果**：无锂环境不启用锂兼容注入
- **关键里程碑**：是

### 步骤 3：构建验证
- **操作对象**：Gradle 任务
- **具体动作**：执行 `compileJava` 与 `remapJar`
- **预期结果**：构建通过
- **关键里程碑**：是

## 预期结果
- 原版与锂兼容逻辑分离，维护风险下降。
- 在无锂环境中，锂专用注入不会生效。
