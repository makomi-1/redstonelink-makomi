# 退役同步与序号外显公共化

生成时间：2026-03-17 12:55:58  
文件名：2026-03-17_125558_退役同步与序号外显公共化.md

## 任务背景
当前“节点退役”路径在命令、批量命令、事件中都复用 `LinkRetireCoordinator`，但退役后未统一触发受影响节点的客户端快照同步，导致近外显可能短时间显示旧连接信息。  
同时远外显与近外显在“颜色、距离、序号文本有效性判断”存在重复实现，需要做最小无行为变更收敛。

## 方案详情

### 现状分析
- 退役入口已统一到 `LinkRetireCoordinator.retireAndSyncWhitelist`，但仅处理数据层退役和白名单清理。
- 近外显连接信息依赖 `PairableNodeBlockEntity` 的客户端快照，若无 `forceSyncToClient` 推送则会滞后。
- `LinkCoreShortCodeRenderer` 与 `LinkSerialHudOverlayRenderer` 均重复处理节点文本与可见距离判断。

### 技术方案
- 在 `LinkRetireCoordinator` 内补齐“退役后受影响节点快照同步”：
  - 退役前采集来源节点在线快照与受影响目标序号集合；
  - 退役后按“来源节点 + 受影响目标节点”执行统一 `forceSyncToClient`。
- 新增客户端公共工具 `LinkSerialOverlayRenderCommon`：
  - 统一节点颜色解析；
  - 统一序号文本判空读取；
  - 统一玩家到节点中心距离判断。
- 远外显与近外显改为调用该公共工具，保持渲染表现不变。

### 影响范围
- 服务端数据协调：`LinkRetireCoordinator`
- 客户端外显渲染：`LinkCoreShortCodeRenderer`、`LinkSerialHudOverlayRenderer`
- 新增公共渲染工具类：`LinkSerialOverlayRenderCommon`

## 原子步骤清单

### 步骤 1：退役后同步补齐
- **操作对象**：`LinkRetireCoordinator`
- **具体动作**：在 `retireAndSyncWhitelist` 内加入受影响节点快照同步逻辑（来源 + 目标）。
- **预期结果**：命令/批量/事件路径统一具备退役后的客户端快照更新。
- **关键里程碑**：是

### 步骤 2：抽取外显公共逻辑
- **操作对象**：`LinkSerialOverlayRenderCommon`（新建）
- **具体动作**：沉淀颜色、序号文本判空、距离判定三个公共函数。
- **预期结果**：远外显/近外显不再重复实现相同判断。
- **关键里程碑**：是

### 步骤 3：渲染器接入公共工具
- **操作对象**：`LinkCoreShortCodeRenderer`、`LinkSerialHudOverlayRenderer`
- **具体动作**：替换重复逻辑为公共函数调用，保持现有视觉行为。
- **预期结果**：实现收敛且功能表现不变。
- **关键里程碑**：是

### 步骤 4：回归验证
- **操作对象**：Gradle 测试任务
- **具体动作**：执行全标签自动化测试并检查结果。
- **预期结果**：变更可编译、测试通过或明确失败点。
- **关键里程碑**：是

## 预期结果
- 退役后近外显连接信息不再依赖后续偶发同步，能够及时刷新。
- 外显相关重复逻辑收敛到单一公共实现，降低维护成本并保持原语义。
