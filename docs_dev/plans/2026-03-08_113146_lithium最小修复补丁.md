# lithium最小修复补丁

生成时间：2026-03-08 11:31:46
文件名：2026-03-08_113146_lithium最小修复补丁.md

## 任务背景
在有 lithium 的外部测试环境中，仍存在“核心粉与原版粉连接后激活锁定”以及“核心粉向上传播原版粉异常”的问题。当前本地无法直接复现锂环境，需要在最小改动前提下先提供可外测的修复补丁。

## 方案详情

### 现状分析
- 已采用“有锂/无锂二选一”注入策略。
- 锂 mixin 已补齐多数通用桥接，但未包含 checkCornerChangeAt 的跨类型桥接。
- 锂快路径注入存在与 lithium 同优先级竞争风险。

### 技术方案
- 仅修改 RedStoneWireBlockLithiumMixin：
  - 将 mixin 优先级调整为 900，降低与 lithium 注入同优先级冲突概率。
  - 新增 checkCornerChangeAt 的 state.is(this) 重定向桥接，复用 isEquivalentWireState(...)。
- 不改动无锂 mixin、plugin 与 mixins 配置，控制变更面。

### 影响范围
- src/main/java/com/makomi/mixin/RedStoneWireBlockLithiumMixin.java
- 不影响无锂环境行为路径。

## 原子步骤清单

### 步骤 1：调整锂 mixin 优先级
- **操作对象**：RedStoneWireBlockLithiumMixin
- **具体动作**：@Mixin(RedStoneWireBlock.class) 改为 @Mixin(value = RedStoneWireBlock.class, priority = 900)
- **预期结果**：降低与 lithium 注入同优先级冲突风险
- **关键里程碑**：是

### 步骤 2：补齐角落传播桥接
- **操作对象**：RedStoneWireBlockLithiumMixin
- **具体动作**：新增 checkCornerChangeAt 的 @Redirect 到 isEquivalentWireState(...)
- **预期结果**：补齐可能影响上行/角落传播的跨类型识别缺口
- **关键里程碑**：是

### 步骤 3：本地编译验证
- **操作对象**：Gradle 构建
- **具体动作**：执行 compileJava
- **预期结果**：确保补丁可编译
- **关键里程碑**：否

在不扩大改动面的情况下，为有锂场景提供可外测的最小补丁，重点验证“锁定问题”和“向上传播问题”是否缓解。
