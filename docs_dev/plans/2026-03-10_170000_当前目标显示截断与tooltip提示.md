# 当前目标显示截断与tooltip提示

生成时间：2026-03-10 17:00:00  
文件名：2026-03-10_170000_当前目标显示截断与tooltip提示.md

## 任务背景
GUI 中“当前连接目标”列表在目标数量较多时会显示过长，影响界面整洁与可读性。需要在主界面文本与 tooltip 中同时做截断，并在截断处提示剩余数量。

## 方案详情

### 现状分析
- 当前显示由 `AbstractMultiPairingScreen.render` 绘制，文本内容来自 `currentLinksLine(currentTargets)`。
- `LinkPairingScreen` 与 `CorePairingScreen` 直接拼接完整目标列表，缺少截断与 tooltip。

### 技术方案
- 新增 UI 常量：主界面最大宽度、tooltip 最大宽度、tooltip 最大显示条目数（100）。
- 在 `AbstractMultiPairingScreen` 中实现通用的列表截断与“剩余数量”计算逻辑，末尾显示 `…(+N)`。
- 主界面行使用截断后的文本，避免过长。
- 鼠标悬停“当前连接”行时显示 tooltip，tooltip 也进行截断并附带 `…(+N)`。

### 影响范围
- `src/client/java/com/makomi/client/screen/AbstractMultiPairingScreen.java`
- `src/client/java/com/makomi/client/screen/LinkPairingScreen.java`
- `src/client/java/com/makomi/client/screen/CorePairingScreen.java`
- `agents/plans/05_后续迭代与测试路线.md`
