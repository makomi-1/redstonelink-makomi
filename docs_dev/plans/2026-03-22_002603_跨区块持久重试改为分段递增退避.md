# 跨区块持久重试改为分段递增退避

生成时间：2026-03-22 00:26:03
文件名：2026-03-22_002603_跨区块持久重试改为分段递增退避.md

## 任务背景

当前跨区块持久 pending 的重试节奏采用“两段式”模型：前期固定 `initialRetryIntervalTicks`，达到 `persistentBackoffStartThreshold` 后统一切换到 `persistentBackoffTicks`。该模型对长时间未送达的 pending 调节粒度不够，用户要求改为按失败次数分段递增的默认退避表。

## 方案详情

### 现状分析

- `CrossChunkDispatchService` 通过 `resolvePersistentRetryIntervalTicks(int attempts)` 只支持两段式重试间隔。
- `RedstoneLinkConfig` 当前仅提供 `initialRetryIntervalTicks`、`persistentBackoffStartThreshold`、`persistentBackoffTicks` 三个键，无法直接表达多段退避。
- 目标区块 `CHUNK_LOAD` 主动唤醒 pending 的机制已经存在，本轮不改变该机制，只替换“失败次数到重试间隔”的映射。

### 技术方案

- 移除持久 pending 的两段式配置主路径，改为四段默认配置：
  - `1~99` 次失败：`1 tick`
  - `100~499` 次失败：`5 tick`
  - `500~999` 次失败：`20 tick`
  - `>=1000` 次失败：`100 tick`
- 配置层使用显式分段字段表达默认值与可调参数：
  - `crosschunk.retry.stage1.maxAttempts`
  - `crosschunk.retry.stage1.intervalTicks`
  - `crosschunk.retry.stage2.maxAttempts`
  - `crosschunk.retry.stage2.intervalTicks`
  - `crosschunk.retry.stage3.maxAttempts`
  - `crosschunk.retry.stage3.intervalTicks`
  - `crosschunk.retry.stage4.intervalTicks`
- 调度层保留 `nextEligibleTick` 机制与目标区块加载唤醒逻辑，仅把 interval 解析改成按分段配置匹配。
- 不做旧键兼容读取，直接采用新键，保持实现与说明一致。
