# 核心粉active驱动收敛

生成时间：2026-03-18 20:41:09
文件名：2026-03-18_204109_核心粉active驱动收敛.md

## 任务背景
当前核心红石粉已完成“点态模型”收敛，但仍保留 `POWER` 状态用于亮暗材质分支与信号读取。该状态与 `ACTIVE` 语义存在重复，增加维护面。目标是改为由 `ACTIVE` 直接驱动亮暗材质与红石输出，并继续做最小范围收敛。

## 方案详情

### 现状分析
- 核心粉方块状态当前为 `POWER + SUPPORT_FACE + ACTIVE`，其中 `POWER` 由 `ACTIVE` 间接同步。
- blockstate 仍按 `power=0 / 1..15` 选择 `dot / dot_bright` 模型。
- 存在孤立模型别名 `models/block/link_redstone_dust_core.json`，当前链路未直接引用。

### 技术方案
- 删除 `POWER` 状态字段，仅保留 `SUPPORT_FACE + ACTIVE`。
- 将信号输出改为按 `ACTIVE` 返回 `coreOutputPower()`。
- 将材质分支改为按 `active=true/false`。
- 粒子触发条件改为 `ACTIVE`，粒子强度继续参考 `coreOutputPower()`。
- 删除未引用的 `models/block/link_redstone_dust_core.json`。

### 影响范围
- `src/main/java/com/makomi/block/LinkRedstoneDustCoreBlock.java`
- `src/main/java/com/makomi/block/entity/LinkRedstoneDustCoreBlockEntity.java`
- `src/main/resources/assets/redstonelink/blockstates/link_redstone_dust_core.json`
- `src/main/resources/assets/redstonelink/models/block/link_redstone_dust_core.json`（删除）

## 原子步骤清单

### 步骤 1：方块状态收敛
- **操作对象**：`LinkRedstoneDustCoreBlock`
- **具体动作**：移除 `POWER` 属性与相关同步逻辑，保留 `SUPPORT_FACE + ACTIVE`。
- **预期结果**：状态语义单一，方块状态数量下降。
- **关键里程碑**：是

### 步骤 2：信号与粒子逻辑对齐
- **操作对象**：`LinkRedstoneDustCoreBlock`
- **具体动作**：`getSignal/getDirectSignal` 与 `animateTick` 改为基于 `ACTIVE`。
- **预期结果**：行为保持不变，逻辑路径更直接。
- **关键里程碑**：是

### 步骤 3：blockstate材质分支切换
- **操作对象**：`link_redstone_dust_core.json`
- **具体动作**：所有 `power` 条件改为 `active` 条件。
- **预期结果**：模型亮暗由激活态直接驱动。
- **关键里程碑**：是

### 步骤 4：资源别名清理
- **操作对象**：`models/block/link_redstone_dust_core.json`
- **具体动作**：删除未引用的模型别名文件。
- **预期结果**：减少冗余资源与误导引用。
- **关键里程碑**：否

### 步骤 5：构建与测试验证
- **操作对象**：Gradle 验证链路
- **具体动作**：执行编译与全标签测试门禁。
- **预期结果**：确认改动无编译/行为回归。
- **关键里程碑**：是

## 预期结果
- 核心红石粉亮暗与输出路径统一为 `ACTIVE` 驱动。
- 减少状态与资源冗余，后续维护点更少。
