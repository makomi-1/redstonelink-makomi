# 近外显连接与位置缩放懒加载

生成时间：2026-03-17 11:22:22
文件名：2026-03-17_112222_近外显连接与位置缩放懒加载.md

## 任务背景
近外显当前每帧都会重建三行文案（含“当前连接”格式化）并重复计算位置缩放比例。用户要求同时为“连接信息”和“近显位置缩放”引入懒加载。

## 方案详情

### 现状分析
- 连接信息：每帧执行 `buildCurrentLinksText`，会触发结构化表达式构建与宽度裁剪。
- 位置缩放：每帧按窗口宽高计算 `min(width/2560, height/1440)`。

### 技术方案
- 在 `LinkSerialHudOverlayRenderer` 内新增近外显文本缓存：
  - 命中维度+坐标、语言签名、序号、激活状态、命中方块、连接快照引用、字体对象标识全部不变时，直接复用上次三行文本。
- 在同文件新增位置缩放缓存：
  - 窗口宽高不变时复用上次 `positionScale`，仅在尺寸变化时重算。

### 影响范围
- 仅影响客户端近外显渲染路径。
- 不改变外显文本内容、颜色语义与显示规则。

## 原子步骤清单

### 步骤 1：连接信息懒加载
- **操作对象**：`src/client/java/com/makomi/client/render/LinkSerialHudOverlayRenderer.java`
- **具体动作**：新增缓存键与缓存命中判定，未变化时复用三行文本
- **预期结果**：减少每帧格式化开销
- **关键里程碑**：是

### 步骤 2：位置缩放懒加载
- **操作对象**：`src/client/java/com/makomi/client/render/LinkSerialHudOverlayRenderer.java`
- **具体动作**：按窗口宽高缓存 `positionScale`
- **预期结果**：窗口尺寸稳定时不重复计算
- **关键里程碑**：是

### 步骤 3：回归门禁
- **操作对象**：Gradle 全标签测试
- **具体动作**：执行 `.\gradlew.bat test testIntegration testClient testSlow testApiLegacy --no-daemon`
- **预期结果**：无回归失败
- **关键里程碑**：是

## 预期结果
近外显在视觉效果不变前提下，减少常态帧的字符串构建与比例计算，降低渲染阶段不必要开销。
