# 核心红石粉物品半透明修复

## 任务背景
当前“连接红石粉核心”在物品栏与手持场景未体现贴图半透明效果。透明变种同类物品模型已具备半透明渲染声明，可作为直接参考路径。

## 方案详情

### 现状分析
- 非透明变种物品模型 `link_redstone_dust_core.json` 仅声明 `parent + textures`，缺少 `render_type`。
- 透明变种物品模型 `link_redstone_dust_core_transparent.json` 已声明 `"render_type": "minecraft:translucent"`。
- 物品栏与手持渲染均走物品模型路径，因此优先在该 JSON 补齐渲染类型即可。

### 技术方案
- 仅修改 `src/main/resources/assets/redstonelink/models/item/link_redstone_dust_core.json`：
  - 新增 `"render_type": "minecraft:translucent"`。
- 不修改任何贴图资源，不调整方块渲染层与方块状态。

### 影响范围
- 影响对象：`link_redstone_dust_core` 物品在 GUI 与手持的透明混合表现。
- 不影响对象：世界内方块渲染逻辑、贴图资源文件、其它物品模型。

## 原子步骤清单

- **具体动作**：新增 `render_type=minecraft:translucent`
- **预期结果**：物品栏与手持场景启用半透明混合
