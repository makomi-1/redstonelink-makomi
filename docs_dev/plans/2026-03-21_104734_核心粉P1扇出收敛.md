# 核心粉 P1 扇出收敛

生成时间：2026-03-21 10:47:34
文件名：2026-03-21_104734_核心粉P1扇出收敛.md

## 任务背景
在 1:900 同步并发场景中，核心粉路径的 `getChunk` 与邻居更新热点明显高于核心块。当前目标是先完成 P1 低风险优化，优先减少无效邻居通知与未加载区块路径进入。

## 方案详情

### 现状分析
- 核心粉与核心块实体都通过 `NeighborFanoutUtil` 做中心+六方向扇出。
- 当前扇出工具未做“目标位置已加载”守卫，可能在通知中触发无效区块访问链路。
- 实体侧已有时间粒度去重守卫，但扇出工具层未形成统一“时间粒度 + 输出值 + 位置”的防重复保障。

### 技术方案
- 在 `NeighborFanoutUtil` 增加：
  - 非阻塞已加载守卫（仅对已加载且在世界边界内的位置执行通知）。
  - 时间粒度扇出去重入口（参数包含 `timeTick/timeSlot/active/resolvedPower`）。
- 在核心块/核心粉实体改为调用带时间粒度参数的扇出入口。
- 在 `ActivatableTargetBlockEntity` 增加受保护读取方法，暴露当前扇出所需时间粒度键。
- 补充契约测试，校验核心块/核心粉对齐到同一策略，并校验工具层具备已加载守卫与时间粒度去重入口。

### 影响范围
- `src/main/java/com/makomi/util/NeighborFanoutUtil.java`
- `src/main/java/com/makomi/block/entity/ActivatableTargetBlockEntity.java`
- `src/main/java/com/makomi/block/entity/LinkCoreBlockEntity.java`
- `src/main/java/com/makomi/block/entity/LinkRedstoneDustCoreBlockEntity.java`
- `src/test/java/com/makomi/block/LinkRedstoneDustCoreStateContractTest.java`

- **预期结果**：扇出工具可在不改变语义前提下减少无效区块通知。
