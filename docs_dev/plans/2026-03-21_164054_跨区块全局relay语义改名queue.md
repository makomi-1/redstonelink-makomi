# 跨区块全局 relay 语义改名 queue

生成时间：2026-03-21 16:40:54
文件名：2026-03-21_164054_跨区块全局relay语义改名queue.md

## 任务背景
当前跨区块配置中，`pulse/toggle` 已经拥有各自独立的普通 TTL relay 配置，但全局“持久派发队列总开关/通用 TTL”仍沿用 `relay` 命名，容易与事件级 relay 混淆。  
本次目标是将全局层语义统一改为 `queue`，并明确不再兼容读取旧 `relay` 配置键。

## 方案详情

### 现状分析
- `RedstoneLinkConfig` 中全局字段、访问器、模板配置仍使用 `relayEnabled` / `relayExpireTicks`。
- `CrossChunkDispatchService` 同时存在“事件级 relayEnabled”和“全局 relayEnabled”两层含义，阅读成本较高。
- `使用说明` 中也仍以 `crosschunk.relay.enabled` / `crosschunk.relayExpireTicks` 描述全局持久队列。

### 技术方案
- 将全局配置键改为：
- `crosschunk.queue.enabled`
- `crosschunk.queue.defaultTtlTicks`
- 内部字段与访问器同步改名为 `queue` 语义：
- `queueEnabled`
- `queueDefaultTtlTicks`
- `crossChunkQueueEnabled()`
- `crossChunkQueueDefaultTtlTicks()`
- `CrossChunkDispatchService` 中全局总闸门变量同步改名为 `queueEnabled`。
- 事件层 `crosschunk.activation.*.relay.*` 保持不变。
- 不兼容读取旧 `crosschunk.relay.*` 键，直接按新键解析与输出。

### 影响范围
- 配置解析与默认模板
- 跨区块派发服务调用点
- 单元测试断言与反射访问
- 使用说明文档

### 语义对齐说明
- 命名继续保持 `triggerSource/core` 基线，无旧术语新增泄漏。
- 输入/输出方向不变，仍为 `triggerSource -> core`。
- 与 `LinkNodeType.TRIGGER_SOURCE/CORE` 映射保持一致。
- 本次不做兼容层，属于直接切换配置命名。
- 残余风险：旧配置文件若仍保留 `crosschunk.relay.*`，升级后将回退到新默认值。
