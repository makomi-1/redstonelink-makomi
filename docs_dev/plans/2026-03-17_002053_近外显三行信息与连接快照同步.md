# 近外显三行信息与连接快照同步

生成时间：2026-03-17 00:20:53
文件名：2026-03-17_002053_近外显三行信息与连接快照同步.md

## 任务背景
用户要求近外显改为三行信息：
1. `[物品名前缀]序号`
2. `激活状态`（ON/OFF）
3. `当前连接`（参考已有 GUI tooltip 的结构化展示方式）

## 方案详情

### 现状分析
- 近外显当前仅显示单行文本，无法承载状态和连接详情。
- 客户端近外显直接读取方块实体数据，但方块实体当前未同步“当前连接列表”到客户端。
- GUI 的“当前连接”格式化已由 `SerialDisplayFormatUtil` 统一实现。

### 技术方案
- 在 `PairableNodeBlockEntity` 增加“当前连接快照”字段，并通过方块实体更新包同步到客户端。
- 在 `set_links` 修改链路后，对来源和受影响目标节点触发一次客户端同步，保证近外显读取到最新连接。
- 重构 `LinkSerialHudOverlayRenderer` 为三行绘制：
  - 第一行：`[物品名]序号`
  - 第二行：`ON/OFF`
  - 第三行：按 `SerialDisplayFormatUtil` 同规则输出的结构化连接文本。

### 影响范围
- `src/main/java/com/makomi/block/entity/PairableNodeBlockEntity.java`
- `src/main/java/com/makomi/command/ModCommands.java`
- `src/client/java/com/makomi/client/render/LinkSerialHudOverlayRenderer.java`
- `docs_dev/使用说明.md`

## 原子步骤清单

### 步骤 1：同步方块实体“当前连接”快照
- **操作对象**：`PairableNodeBlockEntity`
- **具体动作**：新增连接快照字段与 NBT 读写；服务端写入时从 `LinkSavedData` 读取并排序后下发客户端。
- **预期结果**：客户端可直接读取节点当前连接列表。
- **关键里程碑**：是

### 步骤 2：链路变更后强制刷新相关节点客户端快照
- **操作对象**：`ModCommands`
- **具体动作**：在 `set_links` 后对来源节点和受影响目标节点执行一次方块实体同步。
- **预期结果**：近外显连接信息在配对修改后及时更新。
- **关键里程碑**：是

### 步骤 3：近外显改为三行并复用结构化连接格式
- **操作对象**：`LinkSerialHudOverlayRenderer`
- **具体动作**：多行布局渲染、第一行物品名前缀、第二行 ON/OFF、第三行结构化连接文本。
- **预期结果**：近外显满足三行语义要求并保持可读性。
- **关键里程碑**：是

### 步骤 4：同步使用说明
- **操作对象**：`docs_dev/使用说明.md`
- **具体动作**：更新近外显显示格式说明。
- **预期结果**：文档与实现一致。
- **关键里程碑**：否

## 预期结果
- 近外显呈现完整三行信息，不改变现有四态开关语义。
- 当前连接展示格式与 GUI tooltip 保持同源结构化规则。
- 链路修改后近外显连接信息可及时刷新。
