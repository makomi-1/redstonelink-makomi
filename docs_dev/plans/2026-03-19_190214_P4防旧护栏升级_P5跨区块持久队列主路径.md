# P4 防旧护栏升级 + P5 跨区块持久队列主路径

生成时间：2026-03-19 19:02:14
文件名：2026-03-19_190214_P4防旧护栏升级_P5跨区块持久队列主路径.md

## 任务背景
- 当前跨区块派发主路径仍依赖运行期内存队列（`CrossChunkDispatchService.pendingByKey`）。
- 现有逻辑有 TTL 过期保护，但缺少“按 key 单调版本拒旧”的硬护栏。
- 目标是落地 `P4/P5`：防旧护栏升级 + 持久队列替代旧 relay 主路径。

## 方案详情

### 现状分析
- 入队去重已是“同 key 覆盖最新”，但队列仅在内存，重启后无法续跑。
- 投递时未维护 `lastAcceptedVersionByKey`，乱序/迟到包缺少版本拒绝。
- 停服会清空运行态队列，无法形成跨重启的调度延续。

### 技术方案
1. 新增 `CrossChunkDispatchQueueSavedData`（SavedData）  
   - 持久化 `pendingByKey`（同 key upsert）  
   - 持久化 `lastAcceptedVersionByKey`（防旧主护栏）  
   - 持久化 `maxIssuedVersionByKey`（单调版本分配器）
2. `CrossChunkDispatchService` 接入持久队列为主路径  
   - 入队：写入 SavedData（替代内存主队列）  
   - 出队：每 tick 从 SavedData 快照处理  
   - 拒旧：投递前/投递时检查 `version <= lastAcceptedVersion` 直接丢弃  
   - 过期：`expireTick <= now` 丢弃（时间护栏）
3. 运行态内存仅保留索引型状态  
   - 强制加载票据、每 tick 限流计数继续保留内存  
   - 不再保存并行语义队列
4. 停服策略  
   - 保留 pending 持久化数据，重启后续跑  
   - 仅清理 transient/resident 票据与内存限流状态
