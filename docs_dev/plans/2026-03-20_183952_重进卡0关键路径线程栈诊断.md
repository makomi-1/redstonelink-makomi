# 重进卡0关键路径线程栈诊断

生成时间：2026-03-20 18:39:52  
文件名：2026-03-20_183952_重进卡0关键路径线程栈诊断.md

## 任务背景
- 现象：第二次进入世界时卡在 `Preparing spawn area: 0%`。
- 已知：`LinkSavedData.load` 已完成且耗时极低，但未进入 `SERVER_STARTED`。
- 目标：定位 `SERVER_STARTING -> SERVER_STARTED` 之间的阻塞点。

## 方案详情

### 现状分析
- 当前已有生命周期门控和启动心跳日志，但“未进入 SERVER_STARTED”时缺少线程级证据。
- 仅凭阶段日志无法判断是哪个调用链阻塞主线程。

### 技术方案
- 在 `LinkNodeLifecycleDispatchEvents` 增加启动卡住 watchdog（仅诊断）：
  1. `SERVER_STARTING` 记录墙钟起点；
  2. 若超过阈值仍未进入 `SERVER_STARTED`，自动打印“服务器线程栈 + 线程状态摘要”（单次）；
  3. `SERVER_STARTED` 打印入口/出口配对日志并停止 watchdog 观察。
- 在 `LinkSavedData.get` 增加“entry/exit 配对日志（限次）”，补充 DataStorage 边界观测。
- 诊断输出默认对运行时生效，对 Gradle 测试进程静默，避免影响慢测。

### 影响范围
- `src/main/java/com/makomi/data/LinkNodeLifecycleDispatchEvents.java`
- `src/main/java/com/makomi/data/LinkSavedData.java`

### 语义对齐说明
- 命名与方向：不改 `triggerSource/core` 语义，不改来源/目标写入规则。
- API 约束：不新增 `com.makomi.api.v1.*` 依赖。
- 兼容边界：仅新增诊断日志与 watchdog，不改存档结构和业务判定。
- 残余风险：日志量上升，已通过“超时单次抓取 + 限次日志 + 测试进程静默”控制。
