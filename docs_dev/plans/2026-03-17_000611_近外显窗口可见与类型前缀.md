# 近外显窗口可见与类型前缀

生成时间：2026-03-17 00:06:11
文件名：2026-03-17_000611_近外显窗口可见与类型前缀.md

## 任务背景
用户反馈近外显在窗口化时不可见，且希望近外显文本增加类型前缀 `[]`，以提升可读性和语义辨识度。

## 方案详情

### 现状分析
- 近外显 Y 偏移常量过大，导致在较低窗口高度下文本可能被推到可视区域外。
- 近外显目前仅显示序号，不包含节点类型前缀。

### 技术方案
- 保留“偏移量常量”的控制方式，同时引入可视范围夹紧，确保不同分辨率/窗口模式下都在屏幕内显示。
- 在近外显文本前拼接类型前缀：`[core]` / `[triggerSource]`。

### 影响范围
- `src/client/java/com/makomi/client/render/LinkSerialHudOverlayRenderer.java`
- `docs_dev/使用说明.md`

## 原子步骤清单

### 步骤 1：修复近外显窗口可见性
- **操作对象**：`LinkSerialHudOverlayRenderer`
- **具体动作**：调整默认下移值并对绘制中心 Y 做可视区域夹紧。
- **预期结果**：窗口化与全屏模式都可稳定看到近外显。
- **关键里程碑**：是

### 步骤 2：补充近外显类型前缀
- **操作对象**：`LinkSerialHudOverlayRenderer`
- **具体动作**：按节点类型生成前缀并拼接显示文本。
- **预期结果**：近外显文本形如 `[core]12,345` / `[triggerSource]12,345`。
- **关键里程碑**：是

### 步骤 3：同步使用说明
- **操作对象**：`docs_dev/使用说明.md`
- **具体动作**：更新近外显显示格式说明。
- **预期结果**：文档与实现一致。
- **关键里程碑**：否

## 预期结果
- 近外显不再受窗口模式影响而丢失显示。
- 近外显文本包含类型前缀，便于快速识别节点类型。
- 现有四态切换和远外显逻辑保持不变。
