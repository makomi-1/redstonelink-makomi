# 移除 trigger 命令别名

生成时间：2026-03-10 10:45:00
文件名：2026-03-10_104500_移除trigger命令别名.md

## 任务背景
用户要求命令语义中仅保留 `triggersource`，不再保留 `trigger` 别名，避免命令入口语义分叉。

## 方案详情

### 现状分析
当前 `ModCommands` 在 `pair_node`、`set_links`、`retire` 下同时注册了 `trigger` 与 `triggersource`。同时 `parseNodeTypeArg` 也接受 `trigger`，导致命令入口与语义规范不一致。

### 技术方案
采用最小改动路径：
1. 删除命令树中的 `trigger` 分支。
2. 解析层移除 `trigger` 支持，仅保留 `triggersource`（兼容 `trigger_source`）与既有 `button/core`。
3. 同步 README 与错误提示文案，确保文档与运行时一致。

### 影响范围
- `src/main/java/com/makomi/command/ModCommands.java`
- `src/main/resources/assets/redstonelink/lang/zh_cn.json`
- `src/main/resources/assets/redstonelink/lang/en_us.json`
- `README.md`
- `agents/plans/05_后续迭代与测试路线.md`

- **操作对象**：`ModCommands.register()`
- **具体动作**：删除 `pair_node/set_links/retire` 下的 `Commands.literal("trigger")` 分支。
- **预期结果**：命令入口仅保留 `triggersource` 与原有 `button/core`。

- **操作对象**：`parseNodeTypeArg`

- **操作对象**：README 与语言包
