# 批量输入简写统一

## 任务背景
当前命令端与 GUI 端的目标序号输入仅支持逗号/空白分隔，不支持区间段与统一批量简写。需要改为统一格式 `1:100/1/1000/901:1903/`，并支持混合输入、重复提醒、最终去重，同时保证该解析能力可复用到后续入口。

## 方案详情

### 现状分析
- 命令端 `set_links` 使用 `targets` 字符串并在服务端解析。
- GUI 输入框同样通过 `SerialParseUtil` 解析后下发命令。
- 解析工具目前不支持 `A:B` 区间与重复项提示。

### 技术方案
- 扩展 `SerialParseUtil`：
  - 统一分段符为 `/`；
  - 段类型仅支持单值 `N` 与区间 `A:B`；
  - 返回去重结果、无效段、重复段、超限标记。
- 命令端复用新解析结果：
  - 无效输入/超限继续失败；
  - 重复输入仅提醒，不阻断；
  - 写入时基于去重后集合。
- GUI 端复用新解析结果并统一输入提示文本，命令发送参数统一为 `/` 拼接。
- 同步补充中英文文案与使用说明。
