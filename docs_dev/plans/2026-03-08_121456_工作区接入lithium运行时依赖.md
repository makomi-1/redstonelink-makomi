## 任务背景
用户需要在当前开发工作区本地测试 Lithium，并反馈直接将 jar 放入 mods 目录时报错，怀疑与 Fabric API 使用方式有关。

## 方案详情

### 现状分析
- 项目已通过 modImplementation 接入 Fabric API，接法正确。
- 本地测试建议将 Lithium 作为 modRuntimeOnly 运行时依赖接入，避免多目录重复加载。

### 技术方案
- 在 uild.gradle 新增 Modrinth Maven 仓库（仅允许 maven.modrinth 组）。
- 在 dependencies 增加 modRuntimeOnly "maven.modrinth:lithium:"。
- 在 gradle.properties 增加 lithium_version=mc1.21.1-0.15.2-fabric。

### 影响范围
- uild.gradle
- gradle.properties
