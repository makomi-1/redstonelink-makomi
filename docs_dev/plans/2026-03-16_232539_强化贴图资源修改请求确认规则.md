# 强化贴图资源修改请求确认规则

生成时间：2026-03-16 23:25:39
文件名：2026-03-16_232539_强化贴图资源修改请求确认规则.md

## 任务背景
用户要求强化项目执行规则，明确将贴图资源纳入“修改请求确认”管控，避免在未确认场景下直接改动贴图文件。

## 方案详情

### 现状分析
- `AGENTS.md` 已对“代码/配置/资源改动”要求方案确认，但未单独把贴图资源列为高风险资源并做独立门禁。
- `skills/docs-dev-docflow/SKILL.md` 也未明确贴图资源改动前置确认与禁止直接写入约束。

### 技术方案
- 在 `AGENTS.md` 增加“贴图资源改动管控（强制）”小节：
  - 明确受控路径与文件类型（`assets/**/textures/**` 下贴图相关文件）。
  - 强制“修改请求确认”要素（目标文件、目的、方式、回退方案）。
  - 未确认前禁止任何贴图写操作与覆盖回退操作。
  - 若用户要求“只改渲染方式不改贴图”，必须明确贴图为禁止修改范围。
- 在 `skills/docs-dev-docflow/SKILL.md` 同步门禁：
  - 在“总体原则”加入贴图改动必须先确认。
  - 在“更新前检查清单”加入贴图确认状态检查项。
  - 在 `docs_dev/plans/` 条目补充“贴图资源确认状态（如命中）”记录要求。

### 影响范围
- `AGENTS.md`
- `skills/docs-dev-docflow/SKILL.md`

- **关键里程碑**：是
