# 物品栏当前连接目标悬停截断

生成时间：2026-03-10 22:17:38
文件名：2026-03-10_221738_物品栏当前连接目标悬停截断.md

## 任务背景
物品栏悬停提示中的“当前连接”文本直接拼接完整目标列表，数量过多时会影响可读性。需要按最大字符数截断，并在截断处显示 `…(+N)`，且尽量复用现有 GUI 截断风格。

## 方案详情

### 现状分析
- `LinkerItem` 与 `PairableBlockItem` 的 tooltip 直接拼接完整目标列表。
- GUI 端已有“当前连接”截断与 `…(+N)` 提示逻辑，但按像素宽度/条目数，不适用于物品栏 tooltip 的字符数规则。

### 技术方案
- 在 `com.makomi.item` 包内新增轻量工具类，提供按最大字符数截断并追加 `…(+N)` 的构建方法。
- `LinkerItem` 与 `PairableBlockItem` 统一调用该工具类，最大字符数固定为 48。

### 影响范围
- `src/main/java/com/makomi/item/TooltipTextTruncateUtil.java`
- `src/main/java/com/makomi/item/LinkerItem.java`
- `src/main/java/com/makomi/item/PairableBlockItem.java`

## 原子步骤清单

### 步骤 1：新增字符数截断工具类
- **操作对象**：`TooltipTextTruncateUtil`
- **具体动作**：提供按最大字符数拼接与截断方法，超长时追加 `…(+N)`。
- **预期结果**：可复用的 tooltip 截断工具。
- **关键里程碑**：是

### 步骤 2：接入 tooltip 构建
- **操作对象**：`LinkerItem`、`PairableBlockItem`
- **具体动作**：用工具类替换现有的完整拼接逻辑。
- **预期结果**：物品栏悬停提示按字符数截断并显示 `…(+N)`。
- **关键里程碑**：是

## 预期结果
- 物品栏“当前连接”悬停文本过长时可读性提升。
- `LinkerItem` 与 `PairableBlockItem` 共享统一截断逻辑。
