# lithium 红石锁定兼容修复

生成时间：2026-03-08 10:01:54
文件名：2026-03-08_100154_lithium红石锁定兼容修复.md

## 任务背景
用户反馈：在 Fabric 1.21.1 + Lithium 0.15.2 环境下，顶面核心粉与原版红石粉连接后，激活核心粉会出现“锁定无法切换”。
已有结论显示该现象与原版多 wire 实例下 `shouldSignal` 不共享的历史问题同源，但在 Lithium 的优化链路中复现。

## 方案详情

### 现状分析
- 现有 `RedStoneWireBlockMixin` 已在原版链路处理：
  - `calculateTargetStrength` 的邻居输入重定向过滤 wire-like 反馈。
  - `getWireSignal`、`shouldConnectTo` 做跨类型等价桥接。
- Lithium `mixin.block.redstone_wire` 在 `calculateTargetStrength` 头部直接返回 `getReceivedPower(...)`，绕过原有 `getBestNeighborSignal` 注入链路。
- Lithium 私有方法中 `state.is(this)` 仅识别同一 block 实例，无法把“顶面核心粉子类”识别为 wire-like，从而触发反馈锁定。

### 技术方案
- 在 `RedStoneWireBlockMixin` 增加 Lithium 路径的可选重定向（`require = 0`）：
  - `getReceivedPower(...)` 内 `BlockState#is(Block)` 判定改为调用现有 `isEquivalentWireState(...)`。
  - `getPowerFromSide(...)` 三处 `BlockState#is(Block)` 判定同样桥接为 `isEquivalentWireState(...)`。
  - `getStrongPowerTo(...)` 的 `BlockState#is(Block)` 判定同样桥接为 `isEquivalentWireState(...)`。
- 仅修正“wire 等价判定”，不改 Lithium 与原版的功率计算公式，保持最小行为改动。

### 影响范围
- 代码文件：

  - `agents/plans/05_后续迭代与测试路线.md`
- 验证范围：Java 编译链路（`compileJava`）。
