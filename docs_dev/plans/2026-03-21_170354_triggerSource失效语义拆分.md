# triggerSource 失效语义拆分

生成时间：2026-03-21 17:03:54
文件名：2026-03-21_170354_triggerSource失效语义拆分.md

## 任务背景
当前 `triggerSource` 失效相关路径把“区块卸载临时失效”和“其它真实失效”混在同一个 invalidation 语义里。  
此前为避免 `pulse/toggle` 在区块卸载时被回滚，目标侧已将该 invalidation 收口为“仅清理 sync 贡献”，但这也使得离线、解绑、退役、删除等真实失效场景无法清掉 `pulse/toggle` 贡献，出现残留。

## 方案详情

### 现状分析
- `LinkNodeLifecycleDispatchEvents` 的 chunk unload 会发布来源失效事件。
- `LinkRetireCoordinator` 与 `publishLinkDetached` 等真实断链路径也复用同一失效语义。
- `ActivatableTargetBlockEntity` 当前对该失效事件仅剔除 `sync` 贡献，不再剔除 `pulse/toggle`。

### 技术方案
- 将失效语义拆分为两类：
- `triggerSourceChunkUnloadInvalidation`
  - 仅表示 `triggerSource` 所在区块卸载
  - 仅剔除目标上的 `sync` 贡献并重算
  - 默认关闭，可配置
- `triggerSourceInvalidation`
  - 表示除区块卸载外的其它失效，包括离线、解绑、退役、删除等
  - 剔除目标上的 `toggle/pulse/sync` 贡献并重算
  - 默认开启，可配置
- 生命周期回放的 chunk unload 改发第一类事件。
- 真实断链路径改发第二类事件。
- 离线目标时，持久队列新增对应 dispatch kind，保证目标未加载时也能正确回放清理。

### 影响范围
- 配置解析与模板输出
- delta 事件生产与投影
- 跨区块持久队列 kind
- 目标侧 invalidation 处理
- 测试与使用说明

### 语义对齐说明
- 命名统一使用 `triggerSource/core`。
- 方向保持 `triggerSource -> core`。
- “清理”语义明确为“剔除该来源对目标状态的贡献并重算”，不直接写死目标最终态。
- 与 `LinkNodeType.TRIGGER_SOURCE/CORE` 映射保持一致。
- 本次不修 `pulse` 延迟投递时间键问题。
- 残余风险：旧注释/文档若仍把两类失效混称为 `SOURCE_INVALIDATION`，需要同步更新。

## 预期结果
- `triggerSource` 区块卸载与其它失效具备准确、独立的语义。
- 区块卸载默认不清理 `pulse/toggle`，其它失效默认会清理 `toggle/pulse/sync`。
- 目标状态清理统一走“剔除贡献并重算”，不引入额外状态写死逻辑。
