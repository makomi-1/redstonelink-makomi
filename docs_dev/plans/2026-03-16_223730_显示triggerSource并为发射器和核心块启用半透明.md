# 显示 triggerSource 并为发射器和核心块启用半透明

生成时间：2026-03-16 22:37:30
文件名：2026-03-16_223730_显示triggerSource并为发射器和核心块启用半透明.md

## 任务背景
用户反馈当前只能看到核心粉序号外显，看不到 triggerSource；同时希望发射器与接收核心块在放置后具备半透明显示效果，便于场景区分与观测。

## 方案详情

### 现状分析
- 当前序号外显渲染器仅注册在核心粉与透明核心粉方块实体，triggerSource 相关方块实体未注册该渲染器。
- 当前客户端渲染层仅对透明核心块与透明核心粉使用 `translucent`，发射器与非透明核心块仍为默认实体层渲染。
- 发射器与核心块贴图虽然可承载透明通道，但未统一接入半透明渲染层与半透明纹理资产。

### 技术方案
- 将序号外显渲染器泛化到 `PairableNodeBlockEntity`，在渲染器内统一读取节点类型与序号，继续输出 `[type] serial` 与类型颜色。
- 在客户端初始化中，把该渲染器注册到发射器与核心块对应方块实体类型，补齐 triggerSource 可见性。
- 在客户端渲染层注册中，为发射器与接收核心块设置 `RenderType.translucent()`。
- 为发射器与接收核心块（含亮态）生成半透明贴图资源，保证放置后视觉为半透明。
- 更新 `docs_dev/使用说明.md` 的对应条目，补充显示范围与半透明行为说明。

### 影响范围
- 客户端渲染注册：`src/client/java/com/makomi/RedstoneLinkClient.java`
- 序号外显渲染器：`src/client/java/com/makomi/client/render/LinkCoreShortCodeRenderer.java`
- 方块纹理资源：`src/main/resources/assets/redstonelink/textures/block/*.png`
- 使用说明：`docs_dev/使用说明.md`

## 原子步骤清单

### 步骤 1：泛化序号渲染器
- **操作对象**：`LinkCoreShortCodeRenderer`
- **具体动作**：将泛型上界改为 `PairableNodeBlockEntity`，渲染期基于 `getSerial()` 与 `getLinkNodeType()` 统一绘制文案。
- **预期结果**：渲染器可复用到核心与 triggerSource 节点。
- **关键里程碑**：是

### 步骤 2：补齐方块实体渲染器注册
- **操作对象**：`RedstoneLinkClient`
- **具体动作**：把序号外显渲染器注册到三类发射器与接收核心块（及透明核心块）对应方块实体。
- **预期结果**：triggerSource 节点可见 `[triggerSource]` 外显。
- **关键里程碑**：是

### 步骤 3：启用半透明渲染层
- **操作对象**：`RedstoneLinkClient`
- **具体动作**：对发射器与接收核心块加入 `RenderType.translucent()` 注册。
- **预期结果**：目标方块走半透明渲染通道。
- **关键里程碑**：是

### 步骤 4：调整半透明纹理资源
- **操作对象**：`textures/block` 下发射器与核心块相关 PNG
- **具体动作**：生成半透明 alpha 版本，保持色相与亮态差异。
- **预期结果**：放置后视觉呈现稳定半透明。
- **关键里程碑**：是
