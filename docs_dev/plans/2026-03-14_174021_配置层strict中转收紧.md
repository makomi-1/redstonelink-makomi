# 配置层 strict 中转收紧

生成时间：2026-03-14 17:40:21

## 任务背景
用户要求将配置层 type 解析与命令层保持一致，统一仅接受标准词表 `triggerSource/core`，同时保持大小写兼容，并覆盖 preset 的 `type:serial` 解析入口。

## 方案详情

### 现状分析
- 命令层已切换到 canonical 解析（`tryParseCanonicalType`）。
- 配置层仍通过 `resolveTypeForRole` 使用宽松解析，可能接受 `button`、`trigger_source` 等别名。
- 配置入口包含：
  - `crosschunk.whitelist.sourceTypes` / `crosschunk.whitelist.targetTypes`
  - `crosschunk.preset.<name>.sources/targets` 的 `type:serial`

### 技术方案
- 在语义中转层新增“strict 语义解析”入口：
  - `SemanticRolePolicy.resolveCanonicalType(...)`
  - `LinkNodeSemantics.resolveCanonicalTypeForRole(...)`
- 配置层统一改为调用 strict 入口，继续保留 role/allowedTypes 约束与现有告警策略。
- 保持存档读取路径不变，避免影响历史数据兼容。

### 影响范围
- `src/main/java/com/makomi/data/semantic/SemanticRolePolicy.java`
- `src/main/java/com/makomi/data/LinkNodeSemantics.java`
