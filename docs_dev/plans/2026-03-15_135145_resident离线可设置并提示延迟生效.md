# resident离线可设置并提示延迟生效

生成时间：2026-03-15 13:51:45

## 任务背景
当前 resident 设置在命令层会拦截离线序号，导致“可携带场景下先设置、后上线”的操作不便。  
需求是允许离线对象写入 resident，并明确提示离线期间不会强制加载，待节点重新上线后自动恢复；同时强调 linker 不是在线节点入口。

## 方案详情

### 现状分析
- `whitelist add ... resident` 在命令层做在线硬拦截。
- `whitelist set ... resident confirm` 在批量路径做离线硬拦截。
- 调度层本就按在线节点生成期望 resident 票据，离线条目不会续票。

### 技术方案
- 移除 resident 的在线“失败拦截”。
- 保留 resident 写入行为，但在离线场景输出“已设置，离线暂不生效”的提示。
- 保持调度层逻辑不变，继续由在线状态决定是否续票。
