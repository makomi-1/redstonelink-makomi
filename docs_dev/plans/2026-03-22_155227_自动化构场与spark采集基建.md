# 自动化构场与 spark 采集基建

生成时间：2026-03-22 15:52:27
文件名：2026-03-22_155227_自动化构场与spark采集基建.md

## 任务背景
需要为 RedstoneLink 建立一套“命令自动化构建游戏场景并进行测试与采集”的基础设施，覆盖至少以下方向：
- 连接拓扑：1 对多、多对 1、多对多、混合映射
- 来源类型：单类型、混合类型
- 采集方式：优先接入 spark

目标不是一次性做完整压测平台，而是先落一套最小可运行骨架，后续可以继续扩场景矩阵和驱动方式。

## 方案详情

### 现状分析
- 仓库已经具备自动补序号的节点放置命令、批量覆盖链接命令、批量 activate 命令，可以直接复用。
- 节点方块实体 NBT 中存在稳定的 `Serial` 字段，意味着外部脚本可以通过 `/data get block <pos> Serial` 回读 serial，而不必依赖 GUI 或人工抄号。
- 当前 `audit summary csv` 只能输出总量，不提供按坐标的节点明细，因此“放置后回读 serial -> 建链”的自动化仍需要外部脚本协调。
- `run` 目录下已经存在 spark 运行时依赖与配置，但没有现成的 bench datapack、场景矩阵或 RCON 自动化脚本。

### 技术方案
1. 增加 bench 场景矩阵文件 `tools/bench/matrix.json`
   - 定义默认 spark 命令模板、bench 区域、时间参数
   - 提供 3 个最小样例场景：`sync 1->64`、`toggle 64->1`、`mixed 16->16`
2. 增加 `tools/bench/run-bench.ps1`
   - 内置最小 RCON 客户端
   - 支持列出场景、打印场景、安装 datapack、执行单个场景
   - 支持：放置节点、读取 block entity `Serial`、批量 `link set`、执行 spark 开始/停止/health 命令、写出结果 JSON
3. 增加 `tools/bench/datapack/rl_bench`
   - 提供最小公共函数：`load`、`prepare/common`、`reset/common`
   - 只负责 bench 世界公共准备，不直接承担复杂 serial 解析与建链
4. 增加 `tools/bench/bench.server.properties.example`
   - 提供 dedicated server 场景下的 RCON 配置示例
5. 更新 `docs_dev/使用说明.md`
   - 记录 bench 工具的目录、前置条件和最小执行方式

### 影响范围
- `tools/bench/**`
- `docs_dev/使用说明.md`
