# 时序仲裁稳健修复

生成时间：2026-03-19 22:29:38  
文件名：2026-03-19_222938_时序仲裁稳健修复.md

## 任务背景
当前核心目标节点在收到同步事件后，运行态可能被长期 SYNC 优先级压制，导致后续玩家交互与脉冲/切换事件无法按“后到优先”的时序语义生效。  
本次修复要求在不破坏现有结构真值模型与跨区块持久队列设计的前提下，补齐可扩展时间粒度仲裁能力，并为后续 tick 细分预留扩展点。

## 方案详情

### 现状分析
- 目标端当前生效模式采用常驻优先级（SYNC > PULSE > TOGGLE），缺少跨 tick 的统一时间键仲裁。
- 同 tick 仅有轻量仲裁与合并，跨 tick 没有“后到覆盖先到”的运行态 authority。
- 跨区块队列已有 version 护栏，但未把时间键用于目标端主仲裁。

### 技术方案
- 引入统一时间键 `TimeKey(tick, slot)`，当前 `slot` 固定为 `0`，后续可扩展细分粒度。
- 引入事件元数据 `EventMeta(mode, timeKey, seq)` 与目标端运行态 authority（`authorityMode + authorityTimeKey`）。
- 目标端仲裁规则调整为：
  1. 先按 `TimeKey` 比较新旧；
  2. 同 `TimeKey` 再按固定类间优先级 `SYNC > PULSE > TOGGLE`；
  3. `seq` 仅作为防旧护栏与观测字段，不作为主仲裁。
- 保留现有结构真值：`syncSourceStrengths / pulseUntilGameTime / toggleState`，仅改变派生态解析路径。
- 扩展派发链路，使直接派发与跨区块入队都携带 `tick+slot+seq`。
