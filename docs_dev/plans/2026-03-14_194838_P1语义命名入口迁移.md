# P1语义命名入口迁移

生成时间：2026-03-14 19:48:38

## 任务背景
在 P0 完成语义层命名别名基础上，继续推进 P1：将数据层与网络层补齐“source/target”语义入口，并在调用点逐步接入，保持行为不变。

## 方案详情

### 现状分析
- 数据层仍以 `toggleLink/getLinkedCores/getLinkedButtons` 为主，调用方存在 `BUTTON/CORE` 条件分支。
- 网络层仍以 `openButtonPairing/openCorePairing` 为主，缺少统一的语义化入口。

### 技术方案
- 在 `LinkSavedData` 增加语义化别名：
  - `toggleLinkBySourceType(...)`
  - `getLinkedTargetsBySourceType(...)`
- 在 `PairingNetwork` 增加统一入口：
  - `openPairingBySourceType(...)`
  - 旧入口 `openButtonPairing/openCorePairing` 保留并委托。
- 在命令与物品调用点改用语义化入口，减少调用方 `BUTTON/CORE` 分支噪音。
- 增加测试验证语义化别名与旧方法行为等价。

### 影响范围
- `src/main/java/com/makomi/data/LinkSavedData.java`
- `src/main/java/com/makomi/network/PairingNetwork.java`
- `src/main/java/com/makomi/command/ModCommands.java`
- `src/main/java/com/makomi/command/activate/ActivateCommandRegistry.java`
- `src/main/java/com/makomi/item/PairableBlockItem.java`
