## 任务背景
用户确认将语义中转收紧为一对一映射，要求仅接受 canonical 语义词表 `triggerSource/core`，移除历史别名与宽松回退解析。

## 方案详情

### 现状分析
- `SemanticTypeAliasResolver.tryResolve` 当前仍接受 `button`、`trigger_source`、`trigger-source` 等别名，并包含 `LinkNodeType.values()` 名称回退。
- `resolveCompatibleTypeForRole` 与 `resolveStrictTypeForRole` 在行为上存在“兼容词表”和“严格词表”差异。
- 相关测试用例仍验证旧别名成功解析。

### 技术方案
- 将 `SemanticTypeAliasResolver.tryResolve` 收紧为与 canonical 一致，仅接受 `triggerSource/core`（大小写不敏感）。
- 删除别名解析和枚举名称回退逻辑，确保语义中转一对一。
- 保留现有 API 入口名（如 `resolveCompatibleTypeForRole`），但使其内部行为与 strict 保持一致。
- 同步修正语义测试断言，改为验证旧别名失败、canonical 成功。
