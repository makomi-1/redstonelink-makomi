# 跨区块子分类标题分割线样式调整

生成时间：2026-03-22 09:17:11

## 任务背景

上一轮已将 `crosschunk` 模板拆分为多个子分类，但子分类标题仍使用简单的 `# -- Xxx --` 形式，视觉分隔效果不够明显。用户要求把每个子分类标题改成带分割线的样式。

## 方案详情

### 现状分析

- `crosschunk` 模板块位于 `RedstoneLinkConfig` 默认配置字符串中。
- 子分类标题目前仅为短横线文字，不影响任何解析逻辑。
- 本轮属于纯模板注释样式调整，最小改动应只修改标题文本。

### 技术方案

- 保持 `crosschunk` 模板中的键名、顺序、默认值与注释正文不变。
- 仅把 7 个子分类标题统一改为带分割线的形式，例如：
  - `# ----- [Whitelist & Preset] -----`
- 不调整运行逻辑，不改其它模板区域。

### 影响范围

- `src/main/java/com/makomi/config/RedstoneLinkConfig.java`

## 语义对齐说明

- 术语继续保持 `triggerSource/core`，不引入旧术语。
- 不影响方向语义与 `LinkNodeType.TRIGGER_SOURCE/CORE` 映射。
- 不改配置兼容边界，属于纯注释样式改动。
- 残余风险：无。

## 原子步骤清单

### 步骤 1：调整子分类标题样式
- **操作对象**：`RedstoneLinkConfig`
- **具体动作**：将 `crosschunk` 模板内 7 个子分类标题统一替换为带分割线样式
- **预期结果**：子分类标题视觉上更醒目
- **关键里程碑**：是

### 步骤 2：校验文本与构建
- **操作对象**：修改后的模板文件
- **具体动作**：检查 UTF-8 编码、无替换字符，并执行全量测试
- **预期结果**：确认本次纯注释改动未引入副作用
- **关键里程碑**：否

## 预期结果

`crosschunk` 模板将具备更清晰的分段视觉层次，方便快速扫描定位具体子分类，同时保持现有行为完全不变。
