# 核心粉短码外显最小实现

生成时间：2026-03-16 19:59:45
文件名：2026-03-16_195945_核心粉短码外显最小实现.md

## 任务背景
为了先验证“方块外显序号”的可读性与视觉干扰，先实现短码展示，不引入完整序号悬浮信息与复杂交互。

## 方案详情

### 现状分析
- 核心粉序号已存在于 `PairableNodeBlockEntity`，客户端可通过方块实体同步包读取。
- 当前客户端未注册任何核心粉方块实体渲染器，无法在世界中绘制短码文本。

### 技术方案
- 在客户端新增核心粉专用短码渲染器（BlockEntityRenderer）。
- 短码规则：以序号转 Base36 大写，左侧补零到 8 位，格式化为 `XXXX-XXXX`。
- 仅在有合法序号时渲染，且限制距离（默认 8 格内）降低视觉噪音。
- 仅注册到核心粉与透明核心粉两类方块实体，不影响其它方块。

### 影响范围
- `src/client/java/com/makomi/RedstoneLinkClient.java`
- 新增 `src/client/java/com/makomi/client/render/LinkCoreShortCodeRenderer.java`
- `src/main/java/com/makomi/block/entity/LinkRedstoneDustCoreBlockEntity.java`（提供短码访问方法）

## 原子步骤清单

### 步骤 1：新增短码生成入口
- **操作对象**：`LinkRedstoneDustCoreBlockEntity`
- **具体动作**：增加短码访问方法，输出稳定 `XXXX-XXXX` 格式。
- **预期结果**：渲染层可直接获取短码字符串。
- **关键里程碑**：是

### 步骤 2：新增客户端渲染器
- **操作对象**：`LinkCoreShortCodeRenderer`
- **具体动作**：实现短码文本渲染逻辑（朝向相机、居中、距离裁剪）。
- **预期结果**：世界内可看到短码外显。
- **关键里程碑**：是

### 步骤 3：注册渲染器
- **操作对象**：`RedstoneLinkClient`
- **具体动作**：为核心粉与透明核心粉方块实体注册短码渲染器。
- **预期结果**：客户端启动后渲染生效。
- **关键里程碑**：是

### 步骤 4：回归验证
- **操作对象**：Gradle 测试任务
- **具体动作**：执行 `.\gradlew.bat test testIntegration testClient testSlow testApiLegacy --no-daemon`。
- **预期结果**：改动不引入回归。
- **关键里程碑**：是

## 预期结果
核心粉方块可以在世界中外显简短、稳定、可读的短码，用于快速目测验证效果；后续再按实际观感迭代样式与完整序号展示。
