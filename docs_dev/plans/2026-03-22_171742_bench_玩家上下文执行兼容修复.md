# bench 玩家上下文执行兼容修复

生成时间：2026-03-22 17:17:42
文件名：2026-03-22_171742_bench_玩家上下文执行兼容修复.md

## 任务背景
外部 dedicated server 继续推进到构场阶段后，bench 脚本收到 `[RedstoneLink] This command can only be used by a player.`。排查后确认 `redstonelink place` 与 `redstonelink link set` 都要求玩家命令源，直接从 RCON 控制台执行会失败，随后 serial 回读命中空气方块并连带报错。

## 方案详情

### 现状分析
- `tools/bench/run-bench.ps1` 当前默认将所有 bench 命令直接以 RCON 控制台源执行。
- `redstonelink place`、`redstonelink link set` 在服务端实现中都通过 `source.getPlayer()` 获取玩家，RCON 直调必然失败。
- bench 的 `data get block`、`function rl_bench:*`、`fill` 等世界相关命令如果不与放置命令保持同一执行上下文，还可能出现维度不一致问题。

### 技术方案
- 为 `RunCase` 增加 `-AsPlayer` 参数，表示“bench 运行时借用的在线玩家”。
- 增加统一的玩家上下文包装函数，将 world-sensitive/player-only 命令转换为：
  `execute as <player> at <player> run <command>`
- `reload` 与 `spark` 仍保留控制台执行；构场、serial 回读、建链、audit、驱动命令统一走玩家上下文。
- 当真实 `RunCase` 未提供 `-AsPlayer` 时，提前失败并给出明确提示。

### 影响范围
- `tools/bench/run-bench.ps1`
- `docs_dev/使用说明.md`
