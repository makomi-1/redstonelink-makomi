# P0 待退役队列持久化

生成时间：2026-03-16 14:44:26

## 任务背景
当前 `LinkNodeRetireEvents.PENDING_RETIRES` 仅保存在内存中。  
当服务器重启或异常退出时，尚未到期的待退役任务会丢失，导致节点未按预期退役。

## 方案详情

### 现状分析
- 退役任务入队、取消、到期处理都在 `LinkNodeRetireEvents` 内存结构中执行。
- 生命周期目前只在 `END_SERVER_TICK` 处理任务、`SERVER_STOPPED` 清理内存。
- 缺少持久化镜像与重启恢复流程。

### 技术方案
- 新增独立 `SavedData`：`PendingRetireQueueSavedData`，用于持久化待退役任务最小字段（type、serial、dimension、pos、expireTick）。
- 在 `LinkNodeRetireEvents` 的入队/取消/到期处理点同步写入或删除持久化镜像。
- 在 `SERVER_STARTED` 从 `SavedData` 回放内存队列；在 `SERVER_STOPPING` 先做一次到期兜底处理再清理内存。

### 影响范围
- `src/main/java/com/makomi/data/LinkNodeRetireEvents.java`
- `src/main/java/com/makomi/data/PendingRetireQueueSavedData.java`（新增）
- `src/test/java/com/makomi/data/PendingRetireQueueSavedDataTest.java`（新增）
