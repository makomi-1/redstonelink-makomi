# 跨区块调度整合实现（语义隔离+白名单+命令拆分）

生成时间：2026-03-12 16:11:08

## 任务背景
跨区块联动在目标未加载时会直接跳过，导致关键场景失效；当前跨区块调度尚未落地语义隔离、类型+序号白名单与命令模块化，难以精确控制强制加载与联动策略。

## 方案详情

### 现状分析
- 触发链路在目标区块未加载时直接跳过。
- 目标端已有 SYNC/PULSE/TOGGLE 仲裁与合并语义，需要保持不变。
- 配置仅包含基础项，跨区块策略与命令权限未配置化。
- 命令树仍集中在 `ModCommands`，缺少独立 cross-chunk 子模块。

### 技术方案
- 新增“跨区块调度层”，未加载时进入 Relay Cache 或 Force-Load，已加载时保持原分支不变。
- 调度层在 ServerTick 中派发并清理 TTL/过期项，避免堆积。
- 新增语义工具类，落实 triggerSource/core 隔离与类型+序号白名单。
- 白名单运行态落地 SavedData；配置仅保留权限与只读 preset。
- 命令拆分到 `com.makomi.command.crosschunk.*`，`ModCommands` 只注册入口。

### 影响范围
- 触发链路：`TriggerSourceBlockEntity`
- 调度与持久化：新建 `CrossChunkDispatchService`、`CrossChunkWhitelistSavedData`
- 语义校验：新建 `LinkNodeSemantics`
- 配置：`RedstoneLinkConfig` 及默认配置模板
- 命令：`ModCommands` + `com.makomi.command.crosschunk.*`
- 文案与文档：`lang/*.json`、`docs_dev/使用说明.md`
