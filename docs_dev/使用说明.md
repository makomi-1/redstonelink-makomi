# 正文

## 跨区块接管提示（2026-03-13）
- 触发端在“跨区块接管实际生效”时提示；未接管不提示。
- 配置项（`config/redstonelink-server.properties`）：
1. `crosschunk.notify.enabled`：跨区块提示总开关（默认 `true`）。
2. `crosschunk.notify.mode`：提示模式（`simple` / `detailed`，默认 `simple`）。
- 显示规则：
1. 统一输出来源：`类型+序号`。
2. 强加载目标与中继目标分行展示，每行显示 `类型+序号` 列表。
3. `simple` 每类最多展示 3 条，`detailed` 每类最多展示 50 条，超出部分显示 `(+n)`。

## 遥控器触发链路对齐（2026-03-12）
- 两种遥控器（`redstonelink_toggle_linker` / `redstonelink_pulse_linker`）触发路径已改为复用触发源同链路派发实现。
- 遥控器触发现在与按钮/拉杆一致，目标未加载时同样进入跨区块 relay/forceLoad 调度逻辑。
- 交互方式与命令入口不变（仍为站立右键主手触发、潜行右键打开配对）。

## 跨区块策略组合（2026-03-12）
- 语义约束：
1. `triggerSource` 仅可作为来源（映射 `LinkNodeType.BUTTON`）。
2. `core` 仅可作为目标（映射 `LinkNodeType.CORE`）。
- 核心配置（`config/redstonelink-server.properties`）：
1. `crosschunk.relay.enabled`：中继缓冲总开关。
2. `crosschunk.relayExpireTicks`：中继缓冲过期时长（tick）。
3. `crosschunk.syncSignalTtlTicks`：SYNC=ON 缓冲 TTL（tick）。
4. `crosschunk.forceLoad.enabled`：强制加载总开关。
5. `crosschunk.forceLoad.mode`：`all` / `whitelist`。
6. `crosschunk.forceLoad.ticketTicks`：强制加载票据时长（tick）。
7. `crosschunk.forceLoad.maxPerTick`：每 tick 强制加载上限。
8. `crosschunk.forceLoad.maxPerSourcePerTick`：每来源每 tick 强制加载上限。
9. `crosschunk.whitelist.sourceTypes` / `crosschunk.whitelist.targetTypes`：可参与白名单的类型。
10. `crosschunk.preset.<name>.sources` / `crosschunk.preset.<name>.targets`：只读 preset（`type:serial`）。
- 组合矩阵（未加载目标时）：
1. `relay=true` + `forceLoad=false`：仅中继缓冲。
2. `relay=true` + `forceLoad=true` + `mode=whitelist`：中继 + 白名单强制加载。
3. `relay=true` + `forceLoad=true` + `mode=all`：中继 + 全量强制加载。
4. `relay=false` + `forceLoad=false`：直接跳过。
5. `relay=false` + `forceLoad=true` + `mode=whitelist`：仅白名单强制加载，未命中直接跳过。
6. `relay=false` + `forceLoad=true` + `mode=all`：全量强制加载（不依赖白名单）。

## 跨区块命令
- 命令根：`/redstonelink crosschunk`
1. `whitelist add <role> <type> <serial>`：新增运行态白名单。
2. `whitelist remove <role> <type> <serial>`：移除运行态白名单。
3. `whitelist list <role> <type>`：查看某角色+类型的白名单。
4. `whitelist clear <role> <type>`：清空某角色+类型的白名单。
5. `preset list`：列出只读 preset 名称。
6. `preset show <name>`：显示 preset 的 sources/targets 明细。

## 其他运维命令
1. `node get/list`：查询节点信息与状态。
2. `link get`：查询来源节点当前关联目标列表。
3. `audit summary text/csv`：输出审计汇总。
4. `place dry_run/force`：放置流程预演与强制执行。

## 自动化测试命令
1. `./gradlew test`：稳定核心 `stable-core` 套件（默认门禁）。
2. `./gradlew testIntegration`：`integration` 标签测试。
3. `./gradlew testClient`：`client` 标签测试。
4. `./gradlew testSlow`：`slow` 标签测试。
5. `./gradlew testExtended`：稳定核心 + 扩展测试集合。
6. `./gradlew testCoverageReport`：生成 XML/HTML 覆盖率报告。
7. `./gradlew testCoverage`：执行扩展测试并校验覆盖率阈值。
