# 03 配对界面与命令实现（已按本地代码更新）

## 目标
提供可视化、多目标、可审计的配对流程，并将输入校验与服务器台账强绑定。

## 当前实现
1. 配对 UI 已抽象为基类：
   - `AbstractMultiPairingScreen` 提供统一布局、输入解析、非法输入提示、上限控制（最多 128 个目标）。
   - 子类：`LinkPairingScreen`（按钮端）与 `CorePairingScreen`（核心端）。
   - UI 显示“当前序号 + 当前连接列表”，支持逗号/空格多输入。
2. 打开方式已统一：
   - 物品：潜行右键可开配对 UI。
   - 已放置方块：空手潜行右键可开配对 UI（核心与按钮均支持）。
   - 服务端通过 `PairingNetwork` 下发 S2C payload 打开 UI，并携带当前连接列表。
3. 命令体系已扩展：
   - `redstonelink pair main|off <serial>`（兼容单目标入口）。
   - `redstonelink pair_node button <source_serial> <target_serial>`。
   - `redstonelink set_links button|core <source_serial> [targets]`（多目标替换）。
   - `redstonelink audit`（在线节点/链接统计 + 活跃/退役序号名单）。
   - `redstonelink retire core|button <serial> confirm`（手动退役）。
4. 输入校验与台账约束：
   - 源序号与目标序号都要求“已分配且未退役”。
   - `set_links` 对非法 token、未分配序号、已退役序号分别返回明确提示。
   - 离线目标允许保存链路，但会提示“当前不可交互”。
5. 物品状态同步：
   - 命令执行后会同步刷新玩家物品 `rl_links` 快照，用于 tooltip 显示当前连接。

## 关键结果
- 配对从“单目标写入”升级为“多目标可视化管理”。
- 校验粒度达到“逐项定位无效输入”，便于用户排错。
- 命令、UI、台账三者语义对齐。
