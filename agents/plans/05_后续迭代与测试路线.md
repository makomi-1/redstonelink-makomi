# 05 后续迭代与测试路线（基于 redstonelink_plan.txt）

## 目标
将当前零散待办收敛为可执行迭代计划，确保每一项都有明确实现边界、验证方式和交付顺序。

## 状态总览

### 已完成（与当前代码一致）
1. 红石连接原件已实现并可用于合成改造对象。
2. 三套基础配方已落地（连接原件/核心/按钮）。
3. 核心与按钮在创造页与搜索中可见。
4. 多对多配对、序号台账、退役/审计命令已落地。
5. 流体冲刷移除不再直接退役：核心/按钮 `onRemove` 改为统一 `unregisterNode`，退役仅留给可判定销毁路径。
6. 交互约束细化已完成：已放置核心/按钮需“潜行 + 双手空手（主手空 + 副手空）”打开配对 UI。

### 待实现（来自待办清单）
1. 配方自动化：为同类核心/按钮变体建立可批量生成流程（数据生成器或脚本）。
2. 许可证文件补充（README / LICENSE 一致性检查）。
3. 红石线新节点（可爬墙、吸附、透明变种）设计与实现。
4. 触发器行为统一：对齐 MC 原生激活逻辑并支持交叉变种。
5. 自定义贴图资源补全。
6. 堆叠问题专项治理与回归测试。
7. 自动清理策略完善 + 台账退役回归测试。
8. 配置文件支持（config）与可调参数接入。
9. 服务端专项审查与测试。
10. 完整回归审查与交付验收。
