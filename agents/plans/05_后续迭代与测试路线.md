# 05 后续迭代与测试路线（基于 redstonelink_plan.txt）

## 目标
将当前零散待办收敛为可执行迭代计划，确保每一项都有明确实现边界、验证方式和交付顺序。

## 状态总览

### 已完成（与当前代码一致）
1. 红石连接原件已实现并可用于合成改造对象。
2. 三套基础配方已落地（连接原件/核心/按钮）。
3. 核心与按钮在创造页与搜索中可见。
4. 多对多配对、序号台账、退役/审计命令已落地。

### 待实现（来自待办清单）
1. 流体冲刷场景生命周期修正：被水冲掉不应直接退役（需与“真正销毁”区分）。
2. 交互约束细化：已放置节点是否放宽为“右手空即可”打开 UI（当前为潜行+空手）。
3. 配方自动化：为同类核心/按钮变体建立可批量生成流程（数据生成器或脚本）。
4. 许可证文件补充（README / LICENSE 一致性检查）。
5. 红石线新节点（可爬墙、吸附、透明变种）设计与实现。
6. 触发器行为统一：对齐 MC 原生激活逻辑并支持交叉变种。
7. 自定义贴图资源补全。
8. 堆叠问题专项治理与回归测试。
9. 自动清理策略完善 + 台账退役回归测试。
10. 配置文件支持（config）与可调参数接入。
11. 服务端专项审查与测试。
12. 完整回归审查与交付验收。

## 原子步骤清单

### 步骤 1：生命周期与交互规则收敛
- **操作对象**：`LinkCoreBlock`、`LinkButtonBlock`、`LinkNodeRetireEvents`
- **具体动作**：梳理“卸载/转移/销毁”边界，修正水流冲刷与实体销毁路径；确认 UI 打开手势规则。
- **预期结果**：退役只在可判定销毁时发生；交互行为符合最终约束。
- **关键里程碑**：是

### 步骤 2：配方自动化与变体可扩展
- **操作对象**：`src/client/java/com/makomi/RedstoneLinkDataGenerator.java`、`data/redstonelink/recipe/*`
- **具体动作**：抽象核心/按钮配方模板，支持批量生成；补充生存模式回归用例。
- **预期结果**：新增同类物品时无需手写重复配方 JSON。
- **关键里程碑**：是

### 步骤 3：资源与视觉资产完善
- **操作对象**：`assets/redstonelink/models/*`、`assets/redstonelink/textures/*`、`lang/*`
- **具体动作**：引入自定义贴图并修订模型映射，补齐多语言文案一致性。
- **预期结果**：视觉资源与行为一致，发布可用。
- **关键里程碑**：否

### 步骤 4：红石线节点与触发器体系扩展
- **操作对象**：新方块/方块实体、触发逻辑抽象层
- **具体动作**：实现红石线节点（爬墙/吸附/透明变种）；触发器对齐原生激活判定并支持交叉组合。
- **预期结果**：节点体系从“核心+按钮”扩展到“可组合触发网络”。
- **关键里程碑**：是

### 步骤 5：配置化与服务端质量门禁
- **操作对象**：config 模块、命令校验、服务端测试脚本
- **具体动作**：新增可配置项（如清理策略、交互限制、日志级别）；补齐服务端压力与一致性测试。
- **预期结果**：可运维、可调优、可回归。
- **关键里程碑**：是

### 步骤 6：完整回归与交付验收
- **操作对象**：构建流程、测试清单、发布产物
- **具体动作**：执行功能回归 + 生存回归 + 兼容性回归；输出版本说明与验收结论。
- **预期结果**：形成可发布版本与可复用验收流程。
- **关键里程碑**：是

## 验收标准
1. 生命周期：不会将“非销毁性移除”误判为退役。
2. 交互：核心/按钮 UI 打开规则与文档一致，且无误触发。
3. 配方：自动化生成可覆盖核心/按钮变体且结果可构建。
4. 功能：新增节点与触发器逻辑通过生存模式实测。
5. 质量：服务端审查通过，`./gradlew build` 成功。
