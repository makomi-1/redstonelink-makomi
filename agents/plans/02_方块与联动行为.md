# 02 方块与联动行为实现（已按本地代码更新）

## 目标
完成核心/按钮两类节点的放置、触发、掉落、生命周期与跨节点联动能力。

## 当前实现
1. 方块实体分层已完成：
   - `PairableNodeBlockEntity`：统一序号、最近目标、节点注册/注销/退役、NBT 同步。
   - `ActivatableTargetBlockEntity`：统一可激活目标行为（`TOGGLE/PULSE` 两种激活模式）。
   - `TriggerSourceBlockEntity`：统一触发源行为（读取链路并触发目标）。
2. `LinkCoreBlock + LinkCoreBlockEntity`：
   - 方块属性从红石块复制，显式关闭发光（`lightLevel -> 0`）。
   - `ACTIVE` 为红石输出开关，输出强度为 15/0。
   - 空手潜行右键打开核心配对 UI；普通空手右键执行本地触发切换。
   - 放置时通过 `resolvePlacementSerial` 绑定序号并注册节点。
3. `LinkButtonBlock + LinkButtonBlockEntity`：
   - 基于石按钮外观/按压时序。
   - 红石输出被完全屏蔽（`isSignalSource/getSignal/getDirectSignal` 全部返回无输出）。
   - 按压时触发所有已链接核心（多对多）。
   - 空手潜行右键打开按钮配对 UI。
4. 掉落与生命周期处理：
   - 两类方块掉落时继承节点序号，并写回当前链接快照（`rl_links`）。
   - 方块被流体替换时执行 `retireNode`；普通破坏执行 `unregisterNode`。
   - 掉落物带 `rl_destroy_retire` 标记，后续被真正销毁时可自动退役。
5. 触发可达性策略：
   - 目标序号在线且区块可达时才会触发实际方块实体。
   - 离线或不可达目标保留链路数据，不做强制清理。

## 关键结果
- 核心/按钮共性已抽象到基类，新增同类节点时复用成本低。
- “按钮不输出红石，但可远程驱动核心”已稳定落地。
- 生命周期行为（放置/破坏/销毁）与序号台账联动一致。
