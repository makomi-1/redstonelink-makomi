# RedstoneLink 1.21.1 设计方案

## 1. 任务目标
实现两种可配对方块（通过方块物品放置）：
1. 链路红石核心（改自红石块材质）：
   - 每个物品实例拥有唯一序号。
   - 放置后可切换是否向周围输出红石信号。
   - 不发光。
2. 链路切换按钮（改自石按钮材质）：
   - 每个物品实例拥有唯一序号。
   - 自身不输出红石信号。
   - 按下后切换已配对红石核心的激活状态。

交互要求：两种物品拿在手里右键打开配对界面，按序号设置配对目标。

约束：
- Fabric + Minecraft 1.21.1
- 沿用原版材质，修改显示名称
- 不实现合成配方

## 2. 核心设计

### 2.1 唯一序号分配
- 使用 `SavedData`（世界持久化）维护全局递增计数器 `nextSerial`。
- 物品通过 `DataComponents.CUSTOM_DATA` 保存：
  - `rl_serial`：自身序号
  - `rl_pair`：配对目标序号（0 表示未配对）
- 物品在服务端 `inventoryTick/use/useOn` 中补齐序号，确保最终落地为唯一值。
- 两种物品设置 `stacksTo(1)`，避免同堆叠内无法区分实例。

### 2.2 方块与方块实体
- 链路红石核心：`BaseEntityBlock`
  - 方块状态：`active`（布尔）
  - 红石输出：`active=true` 输出 15，否则 0
  - 右键空手可手动切换激活状态
- 链路切换按钮：`ButtonBlock + EntityBlock`
  - 保留按钮按压外观与时序
  - 重写红石输出相关方法，始终输出 0
  - 在 `press` 事件中读取自身配对序号，查找并切换目标红石核心

### 2.3 配对与查找
- `SavedData` 同时维护 `serial -> (dimension,pos,type)` 映射，用于跨区块定位。
- 方块放置时登记映射，移除时注销映射。
- 按钮触发时按序号定位目标核心并调用切换。
- 核心支持“反向约束”：如果核心也设置了配对序号，则仅接受对应按钮序号切换。

### 2.4 配对界面
- 客户端 `Screen`（文本输入 + 确认/清空）
- 显示当前手持物品的“自身序号/当前配对序号”
- 提交时通过客户端发送命令：
  - `redstonelink pair main <serial>`
  - `redstonelink pair off <serial>`
- 服务端命令校验手持物品类型并写回 `rl_pair`。

## 3. 资源与命名
- 方块模型复用原版纹理：
  - 红石核心 -> `minecraft:block/redstone_block`
  - 切换按钮 -> `minecraft:block/stone_button*`
- 中文/英文名称改为自定义翻译键。
- 不提供 recipe 文件。

## 4. 风险与应对
1. 1.21.1 API 变更风险：
   - 通过本地映射 `javap` + 官方文档核对方法签名。
2. 序号同步一致性：
   - 仅在服务端分配序号；客户端只展示。
3. 目标方块不在已加载区块：
   - 按钮触发时给出失败提示，不强制加载区块。

## 5. 验证标准
- `./gradlew build` 通过。
- 创造模式获取两种物品，均自动获得唯一序号。
- 右键物品可打开界面并设置配对。
- 按钮不向邻居输出红石，但能切换目标核心输出。
- 核心处于关闭状态时不输出红石且无发光。
