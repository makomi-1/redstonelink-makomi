## 1. 命令放置链路与加载期序号初始化风险

- 问题编号：`RISK-CMD-001`
- 现象：
  - 历史测试地图中存在 `Serial <= 0` 的可配对节点时，若在方块实体加载期自动补序号，可能出现进图卡顿、异常掉落到虚空、退出保存时间异常拉长。
- 触发条件：
  - 大量旧节点同时加载。
  - 加载期直接执行 `resolvePlacementSerial -> setLinkData -> sendBlockUpdated`。
- 影响范围：
  - 单机集成服和独立服务端均可能受影响，旧大地图与批量命令放置场景风险更高。

### 暂行策略（当前采用）

1. 不在加载期自动补序号。
2. 序号补齐仅在自定义放置命令路径执行。
3. 自定义放置命令检测到将替换非空气方块时，要求二次确认后执行。
